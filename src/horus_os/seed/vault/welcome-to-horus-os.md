# Welcome to horus-os

This is your notes vault. The files in here are example notes that shipped
with horus-os so the app is not empty on your first run. They are yours to
keep, edit, or delete. Nothing here is special.

## What horus-os is

horus-os is a self-hosted command center for a small team of AI agents. The
agents read and write notes in this vault, run tools, and leave a trace of
everything they do so you can see how each answer was produced.

## Your starter team

A fresh install seeds five generic agents. You can rename them, edit their
personas, add your own, or delete the ones you do not need.

- Coordinator: routes a request to the right specialist and synthesizes the
  results into one answer.
- Engineer: handles code and technical work in small, verifiable steps.
- Researcher: gathers and analyzes information and summarizes it with sources.
- Writer: turns raw material into clear docs, summaries, and content.
- Operator: watches the running system for tasks, schedules, errors, and
  health, and surfaces what needs attention.

Each agent has a persona file (SOUL.md) under the `agents/` folder in this
vault. Open one to see how it is written, then make it your own.

## How to use this vault

- Drop notes anywhere as plain markdown. The agents can read them.
- Use the example notes below as starting templates.
- Delete anything you do not want. This is your space.

## Clearing the example data

When you are ready to start fresh, delete the example notes above and the
`agents/` personas you do not want. Then run your own first prompt and the
demo trace will be joined by real ones.
