<objective>
Land the read/write REST surface and dashboard tab for the v0.5 plugin system. After this plan:

1. `GET /api/plugins`, `GET /api/plugins/{name}`, `POST /api/plugins/{name}/enable`, `POST /api/plugins/{name}/disable`, `POST /api/plugins/{name}/grant`, `DELETE /api/plugins/{name}/grant/{capability}` — all six routes return strict Pydantic-validated JSON.
2. `GET /api/observability/plugins?since=7d|30d` returns per-plugin error_rate + p50/p95 + total_invocations + total_cost_usd, with NULL `plugin_name` rows bucketed under `"horus-os core"`.
3. `/plugins` dashboard tab cloned from the existing `/adapters` tab pattern: plugin tiles with version, declared tools, declared adapters, capability chips, grant modal, enable/disable toggle, author + homepage + issue_tracker hyperlinks rendered from manifest fields only.
4. `/observability` tab gains a fourth panel "by plugin" alongside the existing cost / latency / tools tiles.
5. DASH-5-03 sanitation: every manifest-sourced string in the DOM goes through `escapeHtml` or `textContent`; `<script>alert(1)</script>` in a manifest author field renders as literal text, asserted by a dedicated regression test.

Purpose: close the runtime user-facing surface for v0.5. After Phase 44 (installer CLI) + this phase, a user can install a plugin, grant capabilities from the dashboard, watch its error rate trend, and revoke from the UI without touching the CLI.

Output: the six new routes; the `/api/observability/plugins` route; the new `/plugins` tab; the per-plugin tile inside `/observability`; four new test files. Byte-identical v0.4 observability surface.

## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Plugin author → manifest | A plugin author writes `horus-plugin.toml` (author, homepage, issue_tracker fields) — these become user-visible HTML in the dashboard. |
| HTTP client → /api/plugins/{name}/grant | Dashboard browser POSTs a capability string. Default-deny enforcement lives behind the route (PermissionService.grant requires actor='dashboard' which is hard-coded; the wire contract carries only the capability). |
| Server → SQLite | Per-request `Database(cfg.db_path)` constructions write through audit log + permission rows. CHECK constraints reject typos. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-45-01 | S (Spoofing) | PermissionService.grant actor field | mitigate | Hard-code `actor='dashboard'` in the grant/revoke route handlers; never accept actor from request body. SQLite CHECK constraint on actor IN ('cli','dashboard','system') is the second line of defense. |
| T-45-02 | T (Tampering) | manifest_author / homepage / issue_tracker rendering | mitigate | DASH-5-03 sanitation: every manifest-sourced string reaches DOM via escapeHtml() or textContent. URL fields gated on startsWith('http://')||startsWith('https://') before rendering as `<a href>`. Regression test `test_dashboard_hyperlinks_sanitized.py` asserts the contract at file level. |
| T-45-03 | I (Information disclosure) | /api/plugins error_message field | mitigate | Truncate last_error to first 200 chars; never include trace_id or filesystem paths in the response (the PluginEntry.error_message already follows the Pitfall 6 message-hygiene rule from Phase 43 — this plan inherits that contract, does not weaken it). |
| T-45-04 | D (DoS) | /api/observability/plugins SQL aggregate | mitigate | Use the existing idx_tool_invocations_plugin(plugin_name, created_at) index (Phase 41); window filter via parse_window enforces a sane upper bound (30d). No unbounded scan. |
| T-45-05 | E (EoP) | /api/plugins/{name}/grant body parsing | mitigate | Validate capability against the closed Capability enum BEFORE the SQL UPSERT; 400 on unknown capability. Default-deny preserved. |
| T-45-06 | R (Repudiation) | Capability grants/revokes from dashboard | mitigate | Every grant/revoke writes one row to plugin_capability_grants_log with actor='dashboard' and the manifest_hash; the audit log is append-only (no DELETE in PermissionService). |
| T-45-07 | T | URL injection via manifest homepage | mitigate | `safeUrl(u)` helper requires `startsWith('http://')` or `startsWith('https://')`; `javascript:` / `data:` / relative URLs all fall through to plain-text rendering with no `<a>` tag. |
| T-45-08 | D | /plugins tab polling | accept | 10s setInterval mirrors the existing /adapters pattern at line 191; load is bounded to one app instance. |
