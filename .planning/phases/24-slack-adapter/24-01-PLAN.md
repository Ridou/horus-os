## SlackAdapter shape

```
class SlackAdapter:
    name = "slack"

    def __init__(self):
        self._client = None
        self._sdk_available = False
        self._sdk_error = None
        self._seen_event_ids = collections.OrderedDict()  # LRU dedup

    def bind(self, app, context):
        try:
            import slack_sdk
            self._sdk_available = True
        except ImportError:
            self._sdk_error = "slack-sdk is not installed; pip install 'horus-os[slack]'"
            context.registry.mark_error(self.name, self._sdk_error)
        # routes mount either way; handlers check _sdk_available
        @app.post("/api/adapters/slack/events")
        async def events(request): ...
        @app.post("/api/adapters/slack/commands")
        async def commands(request): ...
        if self._sdk_available:
            context.registry.mark_running(self.name)
```

## HMAC verification

```
def _verify_signature(secret, body, timestamp, signature):
    # replay protection
    try:
        ts = int(timestamp)
    except (TypeError, ValueError):
        return False
    if abs(time.time() - ts) > 300:
        return False
    base = f"v0:{timestamp}:{body.decode('utf-8')}".encode("utf-8")
    expected = "v0=" + hmac.new(secret.encode("utf-8"), base, hashlib.sha256).hexdigest()
    return hmac.compare_digest(expected, signature or "")
```

## Event handler

For `/api/adapters/slack/events`:

1. Read body bytes, headers `X-Slack-Request-Timestamp` and `X-Slack-Signature`
2. Verify signature -> 401 on failure
3. Parse JSON. If `type == "url_verification"`, return `{"challenge": payload["challenge"]}`
4. If `type == "event_callback"`:
   - check `event.event_id` (Slack sometimes nests at `event_id` top-level too; we read the outer `event_id`)
   - dedup via `_seen_event_ids` LRU; cap at 1000
   - inspect `event.type`:
     - `app_mention`: route
     - `message` with `channel_type == "im"` and no `bot_id` and no `subtype == "bot_message"`: route
     - otherwise: 200 OK, no-op
5. Touch registry on a routed event; mark_error on a `run_agent` exception
6. Always return 200 unless signature failed (401) or SDK missing (503)

## Slash command handler

For `/api/adapters/slack/commands`:

1. Read raw body bytes and parse the same HMAC
2. Verify signature -> 401 on failure
3. `urllib.parse.parse_qs(body.decode("utf-8"))` to extract form fields
4. Required fields: `command`, `text`, `channel_id`, `user_id`
5. Run `run_agent(text)` using the configured profile
6. Return `{"response_type": "in_channel", "text": result.text or "(no response)"}`

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Inbound HTTP request -> HMAC verifier | Adversary forges a Slack request; signing secret protects against this |
| `HORUS_OS_SLACK_SIGNING_SECRET` env var | A leak grants the ability to forge any Slack event to this adapter |
| `HORUS_OS_SLACK_BOT_TOKEN` env var | A leak grants posting permissions to every channel the bot is in |
| Inbound Slack message -> `run_agent` | Adversary prompt-injects the agent via DM, mention, or slash command |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-24-01 | Spoofing | Adversary forges a Slack event to trigger the agent | mitigate | HMAC-SHA256 over `v0:{timestamp}:{body}` with `hmac.compare_digest`; 300-second timestamp window for replay protection |
| T-24-02 | Replay | Adversary captures a real Slack request and re-sends it | mitigate | 300-second timestamp drift cap; event_id LRU dedup gives a second layer (Slack itself never re-uses event_ids within the dedup window) |
| T-24-03 | Information Disclosure | Bot token or signing secret printed in logs | mitigate | The adapter never logs either value; exceptions are captured as `type(exc).__name__: str(exc)`. Setup guide instructs operators to rotate on leak |
| T-24-04 | Tampering | Prompt injection in a DM, mention, or slash command | accept | The adapter calls `run_agent` (no tools) for v0.3; per-channel tool gates are a v0.4 backlog item |
| T-24-05 | Denial | Adversary floods slash commands or mentions, exhausting provider quota | accept | v0.3 leaves rate-limiting to the operator; Slack's own per-app limit (1/sec for `chat_postMessage`) plus the provider's quota are the safety net |
| T-24-06 | Elevation of Privilege | Bot token grants access to workspaces the operator did not intend | mitigate | Setup guide tells operators to install the bot only to their own workspace; no per-workspace allowlist in v0.3 |
| T-24-07 | Repudiation | No record of what the bot said on Slack | accept | The dashboard's trace table captures every `run_agent` invocation. Phase 27 will surface this in the UI |
</threat_model>
