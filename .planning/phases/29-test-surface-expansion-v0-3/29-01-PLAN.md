# horus-os Phase 29 Plan 01

## Stubbing patterns to reuse

### Pattern 1: fake entry_points (lifecycle / coexistence)

`tests/test_adapters_lifecycle.py::_stub_entry_points` returns a
canned list of `_FakeEntryPoint(name, target)` for the
`horus_os.adapters` group. Reuse verbatim. For coexistence, pass
real adapter classes in addition to the live webhook entry point.

### Pattern 2: fake google modules (calendar wiring)

`tests/test_adapters_calendar.py::_install_fake_google` installs
fake `google.*` and `googleapiclient.*` modules into `sys.modules`
with configurable token state, list response, and insert response.
Reuse the function shape; the calendar wiring tests import only
the small subset they need.

### Pattern 3: bind-only fake adapters for coexistence

Use minimal fake Discord and Slack adapters that satisfy the
Protocol surface (`name`, `bind`, optional `start`/`stop`) without
loading SDKs. The point of the coexistence test is to confirm the
discover_adapters + create_app + lifespan loop tolerates the four
of them together, not to retest each adapter's internal logic.

### Pattern 4: AdapterRegistry walk (lifecycle arc)

Walk the registry through all transitions directly. No FastAPI
required; the test asserts purely on registry state. This mirrors
the unit-style test_registry_* cases but as one continuous arc.

### Pattern 5: TestClient + on-route touch (toggle+touch cycle)

A LifecycleAdapter whose `bind` mounts a route that calls
`context.registry.touch(self.name)` lets the test hit a real route
to advance the timestamp.
