<!-- Real APIs; do not re-derive. -->

RRF (FEATURES.md Feature 2 "How results merge"): `score = sum(1 / (k + rank_i))` across keyword rank and vector rank; default `k=60`; results deduped by note id (rel_path) before returning. RRF is rank-based, so BM25 integer scores and cosine floats need no normalization.

NotesStore (src/horus_os/memory/notes.py, after 70-02): `search_notes(query, *, limit=20)` currently does case-insensitive substring scoring returning `list[NoteRef]`; `self._vector_index` is the optional VectorIndex; `_ref_for(path)` builds a NoteRef from a resolved Path; `_resolve(rel)` maps a rel_path back to a Path.

VectorIndex (src/horus_os/memory/vector.py, from 70-02): `search(query_vec: list[float], top_k=20) -> list[tuple[rel_path, distance]]`; `is_ready() -> bool`; `backend.embed([query])[0]` produces the query vector; raises `EmbeddingDimensionMismatch` / `EmbeddingModelMissingError` when not usable.

Server wiring (src/horus_os/server/api.py): `_build_default_registry(cfg, notes_store)` registers the notes tools; `notes_store = NotesStore(cfg.notes_dir, on_write=lambda w: _persist_write(db, w))` is built per request around line 712.

CLI wiring (src/horus_os/cli/run_cmd.py): `_build_default_registry(config, notes_store)`; `notes_store = NotesStore(config.notes_dir, on_write=lambda w: _record_note_write(db, w))` around line 74.

Config (after 70-01/70-02): `vector_memory_enabled`, `embedding_model`, `models_path()`, `vectors_path()`.
