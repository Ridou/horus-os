<objective>
Build the on-device embedding foundation for hybrid vector memory: the `[local-memory]`
optional extra, an `EmbeddingBackend` protocol with an `ONNXEmbeddingBackend` that wraps
fastembed ONNX, and the explicit `horus-os memory download-model` opt-in pre-step. The
whole subsystem is gated so nothing downloads on startup or on a note write, and the system
still starts and serves notes offline with the model file absent.

Purpose: Satisfy MEM-05 (on-device embeddings, no network call on a write) and MEM-06
(explicit CLI download, no silent download, offline start). This plan is the gate the rest
of the phase sits behind; the gate is the first thing built, not an afterthought (Pitfall EM-1).
Output: `memory/embeddings.py`, the `memory` CLI command, config fields, and the `[local-memory]` extra.
</objective>

fastembed (from STACK.md, verified against PyPI fastembed 0.8.0):
- `from fastembed import TextEmbedding`
- `TextEmbedding(model_name="BAAI/bge-small-en-v1.5", cache_dir=<str>, local_files_only=<bool>)`
  - `cache_dir` controls where ONNX model files are read/written. Point it at `<data_dir>/models/`.
  - `local_files_only=True` makes construction read only the local cache and never reach the network; absence of the file raises rather than downloading.
- `list(model.embed(["text"]))` yields one numpy float32 array per input. Default model `BAAI/bge-small-en-v1.5` is 384-dimensional.

Existing Config (src/horus_os/config.py), mirror its idioms:
- `@dataclass class Config` with `data_dir: Path`, `db_path: Path`, `notes_dir: Path`, and optional fields with defaults (see `pricing_path: Path | None = None`).

- `_dump_toml(config)` emits the TOML text; only writes a section when the value is non-default.

Existing CLI dispatch (src/horus_os/__main__.py): `build_parser()` adds a subparser per command, sets `set_defaults(func=run_<name>)`, and the agents/plugins commands show the nested-subparser pattern (`add_subparsers(dest="<name>_command")` + `set_defaults(func=run_<name>, <name>_command="<op>")`).
