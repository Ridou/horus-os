<objective>
Make local-provider runs cost-annotated as local zero-cost ("local (free)"), never as a misleading $0.000000 on an unknown cloud model and never flagged pricing_missing. Add a `local` zero-rate sentinel to the bundled `pricing.json` and an explicit local-provider branch in `CostAnnotator.on_event` that sets `cost_usd = 0.0` and `pricing_missing = False`.

Purpose: The existing cost pipeline keys on model name; a local model like `llama3.1:8b` matches nothing, so the unmodified path would set cost_usd=None + pricing_missing=True (LP-2). That is honest for an unknown CLOUD model but wrong for a local run, which is genuinely free. LLM-04 requires local runs to store cost_usd=0.0, distinguished from NULL, so `/costs` reads "local (free)".

Output: a `local` pricing sentinel; a local-cost branch in CostAnnotator; a regression test asserting a local-provider LLMCallEvent annotates to cost_usd == 0.0 without raising.
</objective>
