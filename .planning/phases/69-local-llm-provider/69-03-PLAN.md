<objective>
Add local-provider onboarding to the init wizard (discover models via `GET {base_url}/v1/models`, list them, run a one-token smoke test before saving config) and a `horus-os doctor --local` endpoint probe that reports valid / unreachable and rejects a `0.0.0.0` base URL as invalid (LP-4). Persist a conservative `local_context_window` so the runtime has a per-model budget basis (LP-3).

Purpose: LLM-02 requires the user to discover and select local models during setup with a one-token smoke test; LP-4 requires base-URL validation so users never silently expose their model API to the LAN; LP-3 requires a context-window field with a soft warning basis. The wizard must be driven deterministically by injected stdin/stdout and injected probe/smoke-test callables, matching the existing run_wizard testing idiom.

Output: a local-provider step in run_wizard with injected discovery + smoke-test validators; a `--local` branch in run_doctor with loopback validation; the doctor subparser flag; deterministic wizard and doctor regression tests.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User config base_url to outbound probe | The wizard discovery probe and doctor probe send requests to the configured base_url. A non-loopback base_url exposes the local model API to the LAN; doctor is the validation chokepoint (LP-4). |
| /v1/models and smoke-test responses to wizard prose | Discovered model names are printed to the user; they originate from the local server the user configured (trusted operator), not arbitrary remote input. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-69-05 | Information disclosure | A 0.0.0.0 or LAN base_url exposes the local model API to other machines | mitigate | doctor `_validate_local_base_url` hard-rejects 0.0.0.0 (invalid, nonzero exit) and warns on any non-loopback host; the wizard never suggests 0.0.0.0 and defaults to loopback (LP-4). Pinned by tests/test_doctor_local.py. |
| T-69-06 | Denial of service | A hanging/unreachable local endpoint stalls the wizard or doctor | mitigate | Discovery and probe callables use a short timeout and surface a warm "unreachable" message instead of blocking or raising; the wizard skips the step and exits cleanly on failure. |
| T-69-07 | Information disclosure | Printing a configured bearer token / key for a remote OpenAI-compat endpoint | mitigate | doctor and wizard never print secrets (matches the existing doctor never-print-service-key contract); only base_url, model ids, and reachability are shown. |
</threat_model>
