## DiscordAdapter shape

```
class DiscordAdapter:
    name = "discord"

    def __init__(self) -> None:
        self._client = None
        self._task = None
        self._context = None

    def bind(self, app, context):
        # No HTTP routes; Discord uses the gateway, not FastAPI.
        return None

    async def start(self, context):
        self._context = context
        try:
            import discord
        except ImportError:
            context.registry.mark_error(
                self.name,
                "discord.py is not installed; pip install 'horus-os[discord]'",
            )
            return
        token = os.environ.get(TOKEN_ENV)
        if not token:
            context.registry.mark_error(
                self.name, f"{TOKEN_ENV} is not set"
            )
            return
        intents = discord.Intents.none()
        intents.guilds = True
        intents.guild_messages = True
        intents.dm_messages = True
        intents.message_content = True
        client = discord.Client(intents=intents)
        client.event(self._make_on_message(client))
        self._client = client
        self._task = asyncio.create_task(client.start(token))
        context.registry.mark_running(self.name)

    async def stop(self):
        if self._client is not None:
            try:
                await self._client.close()
            except Exception:
                pass
        if self._task is not None:
            self._task.cancel()
            with contextlib.suppress(BaseException):
                await self._task
```

The `_make_on_message(client)` factory closes over `client` so the
handler can compare `message.author == client.user` without holding
a separate reference.

## on_message handler

```
def _make_on_message(self, client):
    async def on_message(message):
        # Ignore our own messages to avoid feedback loops.
        if message.author == client.user:
            return
        # Determine whether this is a DM or a mention in a guild.
        is_dm = getattr(message, "guild", None) is None
        is_mention = client.user in getattr(message, "mentions", [])
        if not (is_dm or is_mention):
            return
        prompt = _strip_mention(message.content, client.user)
        if not prompt.strip():
            return
        try:
            reply = self._dispatch(prompt)
        except Exception as exc:
            self._context.registry.mark_error(
                self.name, f"{type(exc).__name__}: {exc}"
            )
            reply = f"Sorry, that failed: {type(exc).__name__}"
        for chunk in _chunk(reply, DISCORD_MESSAGE_LIMIT):
            await message.channel.send(chunk)
        self._context.registry.touch(self.name)
    return on_message
```

`_strip_mention(text, user)` removes the leading `<@id>` or `<@!id>`
mention token so the agent does not see Discord's mention syntax in
the prompt.

`_dispatch(prompt)` looks up the agent profile and calls `run_agent`:

```
def _dispatch(self, prompt):
    cfg = self._context.config
    profile_name = os.environ.get(AGENT_ENV, DEFAULT_AGENT)
    db = Database(cfg.db_path)
    profile = db.load_profile(profile_name) if cfg.db_path.exists() else None
    provider = cfg.default_provider
    model = (profile.default_model if profile else None) or _default_model(cfg, provider)
    system_prompt = profile.system_prompt if profile else None
    result = run_agent(
        prompt,
        provider=provider,
        tools=None,
        model=model,
        system_prompt=system_prompt,
    )
    return result.text or "(no response)"
```

`_default_model` is a small helper mirroring the webhook adapter.

## Chunking

```
DISCORD_MESSAGE_LIMIT = 2000

def _chunk(text, limit):
    if not text:
        yield "(no response)"
        return
    for i in range(0, len(text), limit):
        yield text[i : i + limit]
```

The chunker is intentionally byte-naive: Discord counts characters,
not bytes, so slicing the Python string by length is correct as long
as we do not need to preserve markdown boundaries (which v0.3 does
not promise).

## Environment variables

| Env var | Purpose | Default |
|---------|---------|---------|
| `HORUS_OS_DISCORD_TOKEN` | Discord bot token | required |
| `HORUS_OS_DISCORD_AGENT_PROFILE` | Agent profile name to load | `default` |
| `HORUS_OS_DISCORD_RECONNECT_CAP` | Max backoff seconds (passed to `Client.start(reconnect=...)` if the library supports it) | library default |

The reconnect cap is documented in the setup guide; if the
installed `discord.py` version does not expose the knob, the
adapter passes it through best-effort and the library defaults
apply. We do not write our own outer reconnect loop.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `HORUS_OS_DISCORD_TOKEN` env var -> `discord.Client.start(token)` | A leaked token grants full bot privileges in every server the bot is in |
| Inbound Discord message -> `run_agent` | Adversary can prompt-inject the agent via DMs or mentions |
| `on_message` handler -> `Database.load_profile` | A configured profile's `system_prompt` runs against attacker input |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-23-01 | Information Disclosure | Discord bot token printed in logs or exception messages | mitigate | The adapter never logs the token; exceptions are captured as `type(exc).__name__: str(exc)` and the SDK does not include the token in `LoginFailure` strings. The setup guide warns operators to rotate the token if it leaks |
| T-23-02 | Spoofing | Adversary impersonates the bot in a channel | accept | Discord enforces author identity at the gateway; the adapter trusts `message.author` |
| T-23-03 | Tampering | Prompt injection in a DM or mention causes the agent to perform unintended tool calls | accept | The adapter calls `run_agent` (no tools) for v0.3. Phase 27 / v0.4 will add per-channel tool gates. Documented in the setup guide |
| T-23-04 | Denial | Adversary floods the bot with messages, exhausting provider quota | accept | v0.3 leaves rate-limiting to the operator; Discord's own gateway throttle plus the provider's per-key quota are the safety net. A simple per-user cooldown is a v0.4 backlog item |
| T-23-05 | Elevation of Privilege | Bot token grants access to other guilds where the operator did not intend it to act | mitigate | The setup guide tells operators to invite the bot only to guilds they own; the adapter has no per-guild allowlist in v0.3 |
| T-23-06 | Repudiation | No record of what the bot said on Discord | accept | The dashboard's trace table captures every `run_agent` invocation; the response text is stored. Phase 27 will surface this in the UI |
</threat_model>
