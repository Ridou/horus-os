## Stubbing pattern for offline examples

The test suite stubs providers by setting `_anthropic.stream_anthropic_async`
or `_anthropic.call_anthropic` to a fake. We replicate the same pattern in
example scripts but inline (no pytest fixture):

```
from horus_os._providers import _anthropic

async def _fake_stream(prompt, **kwargs):
    for chunk in ["Hello", " from", " a", " stub"]:
        yield chunk

_anthropic.stream_anthropic_async = _fake_stream
```

This keeps `python examples/streaming.py` runnable without network or
keys. The docstring documents how to revert the stub for a live call.

For the delegate example we stub `run_agent_loop` similarly so the
example does not require provider keys.

For the custom adapter example we stub
`horus_os.adapters.base.entry_points` with a fake that returns our
inline adapter class. `create_app(data_dir=tmp_path)` then mounts it.

## Schema migration shape for the migration guide

v0.1 ships SCHEMA_VERSION = 2 (traces, note_writes).

v0.2 lands SCHEMA_VERSION = 4:
- v2 to v3: adds `agent_profiles` table (CREATE TABLE IF NOT EXISTS in SCHEMA_SQL, idempotent on fresh and upgraded databases).
- v3 to v4: adds `parent_trace_id TEXT` and `agent_profile_name TEXT` columns to traces (guarded with `sqlite3.OperationalError` so re-runs are idempotent), and `idx_traces_parent_trace_id`.

`init()` runs the migration on startup. v0.1 trace rows remain readable; the
new columns are null on those rows. v0.1 users who run `horus-os run` once
under v0.2 trigger the migration automatically.

## CLI behavior changes

| Command | v0.1 | v0.2 |
|---------|------|------|
| `run "prompt"` | buffered output | streams by default; `--no-stream` for v0.1 behavior |
| `run --agent NAME` | not available | loads AgentProfile and threads system_prompt + default_model |
| `agents` group | not available | list, show, create, edit, delete |
| `serve` | unchanged | adds `/api/agents`, `/api/chat/stream`, `/api/traces/{id}/children` |

No flags removed. The default streaming output is the only user-visible
behavior change.
