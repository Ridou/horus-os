# Phase 21 Plan 01: v0.2.0 release

## Objective

Cut the v0.2.0 release. Bump versions, rotate CHANGELOG, run the full
test and lint suite, push the release commit, tag, wait for CI to go
green across all install-smoke matrix jobs, then create the public
GitHub Release. Close out the milestone in STATE.md and ROADMAP.md.

## Tasks

### Task 1: Bump version to 0.2.0

Files: `pyproject.toml`, `src/horus_os/__init__.py`

- Edit the line `version = "0.1.0"` in `pyproject.toml` to `"0.2.0"`
- Edit the line `__version__ = "0.1.0"` in `src/horus_os/__init__.py`
  to `"0.2.0"`
- No other file declares the package version

### Task 2: Rotate CHANGELOG.md

Files: `CHANGELOG.md`

- Rename the existing `## [Unreleased]` heading to
  `## [0.2.0] - 2026-05-23`
- Replace the opening paragraph below that heading ("Heading into
  v0.2.0. Phase 21 will replace this section...") with a short release
  blurb
- Insert a fresh empty `## [Unreleased]` placeholder above `[0.1.0]`,
  with stub `### Added` and `### Changed` subsections so contributors
  know where to write
- Leave all existing v0.2 deliverable bullets exactly as written

### Task 3: Verify lint and tests

- `ruff check .` clean
- `ruff format --check .` clean
- `python -m pytest -q` reports 319 passed

### Task 4: Commit version + CHANGELOG

Single commit, subject `release: v0.2.0`, body summarizing the bumps
and the milestone scope. Co-authored by Claude Opus 4.7.

### Task 5: Push and tag

- `git push origin main`
- `git tag -a v0.2.0 -m "horus-os v0.2.0"`
- `git push origin v0.2.0`

### Task 6: Wait for CI green

Poll `gh run list --limit 1` until the run on the release commit
finishes. All twelve install-smoke matrix jobs and the lint and test
jobs must be green. If any fail, diagnose, fix, push a follow-up
commit, repeat. Do not cut the GitHub Release on a red CI.

### Task 7: Create the GitHub Release

```
gh release create v0.2.0 \
  --title "horus-os v0.2.0" \
  --verify-tag \
  --notes "$(cat <<'EOF'
<body derived from CHANGELOG [0.2.0] section, quoted faithfully>

**Migration:** see `docs/MIGRATION-v0.1-to-v0.2.md` for the v0.1 to
v0.2 upgrade notes.

**Examples:** see the `examples/` directory for `multi_agent.py`,
`streaming.py`, and `custom_adapter.py`.
EOF
)"
```

Verify with `gh release view v0.2.0`.

### Task 8: Mark milestone complete

Files: `.planning/STATE.md`, `.planning/ROADMAP.md`

- `STATE.md`: set `status` to `milestone_complete`, set `progress` to
  21 of 21 phases done, update `last_activity`, add a Prior Milestones
  bullet for v0.2
- `ROADMAP.md`: mark Phase 21 `[x]`, set the v0.2 milestone bullet at
  the top to `[x]`, update the progress table to 1 of 1 plans complete
  for Phase 21

Commit as `docs(21): milestone v0.2 complete, advance state`.

### Task 9: Phase summary

Write `.planning/phases/21-v0-2-0-release/21-01-SUMMARY.md` with the
release SHA, tag SHA, Release URL, CI run URL, and any notable
observations. Commit as `docs(21): summary for plan 21-01`. Push.

## Verification

- `git ls-remote origin refs/tags/v0.2.0` shows the tag
- `gh release view v0.2.0` returns the published release with the
  migration guide link in the body
- `grep -E '^## \[0\.2\.0\]' CHANGELOG.md` finds the new heading
- `grep '^version' pyproject.toml` reports `0.2.0`
- `python -c "import horus_os; print(horus_os.__version__)"` reports
  `0.2.0`
- CI on the release commit is fully green
