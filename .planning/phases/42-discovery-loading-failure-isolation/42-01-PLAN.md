| third-party plugin → horus-os process | Plugin module code executes in the same Python interpreter at import time. No OS-level sandboxing in v0.5 (PROJECT.md "Out of Scope" defers to v0.6+). Phase 42's containment is `try/except` at every load-phase boundary. |
| plugin manifest file → manifest validator | Plugin author controls the TOML bytes; the validator must reject malformed input without crashing. Phase 41 already pinned this; Phase 42 inherits. |
| filesystem plugin directory → discover_plugins() | An attacker with write access to `$HOME/.horus-os/plugins/` can drop a plugin that runs at next horus-os startup. Phase 42 contains the attack surface to "what happens at plugin import time" — the user already controls their home directory, so this is a downstream consequence of compromised local state, not a new vector. |
