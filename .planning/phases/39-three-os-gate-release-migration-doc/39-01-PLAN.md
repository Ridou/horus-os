<!-- Concrete contract for scripts/release_gate.py so the executor implements deterministically. -->

CLI shape:
```
python scripts/release_gate.py                    # all checks, exit 0 on full pass
python scripts/release_gate.py --check pricing    # run only the pricing freshness check
python scripts/release_gate.py --check wheel      # run only the wheel-pricing-bundle check
python scripts/release_gate.py --check ci         # run only the CI two-variant smoke presence check
python scripts/release_gate.py --check tests      # run only the pytest pass check
python scripts/release_gate.py --skip-build       # skip the slow wheel build for fast iteration
```

Environment overrides (documented in docs/RELEASE.md):
- `HORUS_OS_PRICING_MAX_AGE_DAYS=14` (default 14, the REL-08 contract)
- `HORUS_OS_RELEASE_GATE_SKIP_TESTS=1` (skip pytest; useful when running INSIDE a test, see tests/test_release_gate.py)
- `HORUS_OS_RELEASE_GATE_SKIP_BUILD=1` (skip wheel build for fast local iteration)

Check 1 (pricing freshness):
- Read `src/horus_os/observability/pricing.json`
- Parse `updated_at` with `date.fromisoformat(...)` (the field is an ISO-8601 date string, see Phase 34 SUMMARY)
- Compute `age_days = (date.today() - updated_at).days`
- Pass when `age_days <= HORUS_OS_PRICING_MAX_AGE_DAYS` (default 14)
- Fail diagnostic: `"pricing.json.updated_at is {age_days} days old (max {max_age}); refresh per docs/RELEASE.md"`

Check 2 (CI two-variant smoke presence):
- Read `.github/workflows/ci.yml` as text
- Assert literal substring `install-smoke-no-otel` is present (grep-equivalent count >= 1)
- Assert literal substring `install-smoke-with-otel` is present (grep-equivalent count >= 1)
- Fail diagnostic: `"two-variant install-smoke matrix missing from .github/workflows/ci.yml; Phase 38 TEST-15 contract violated"`
- This check is STATIC presence-only; it does NOT execute the CI jobs locally (the maintainer reads GitHub Actions for that)

Check 3 (wheel pricing bundle):
- Run `python -m build --wheel --outdir {tmp_dir}` via subprocess; assert exit code 0
- Locate the produced `.whl` (single file under tmp_dir/dist or tmp_dir directly depending on build output)
- Open as zipfile.ZipFile; assert any member name ends with `/observability/pricing.json` (count >= 1)
- Fail diagnostics: `"python -m build failed"` OR `"built wheel does not contain horus_os/observability/pricing.json; package-data wiring broken"`
- Skipped when `HORUS_OS_RELEASE_GATE_SKIP_BUILD=1` or `--skip-build` passed

Check 4 (pytest pass):
- Run `python -m pytest -q` from repo root via subprocess; assert exit code 0
- Capture last 20 lines on failure into the diagnostic
- Skipped when `HORUS_OS_RELEASE_GATE_SKIP_TESTS=1`

Exit semantics:
- Exit 0 ONLY when all enabled checks pass
- Exit 1 when any check fails; print all failure diagnostics (do NOT short-circuit; the maintainer wants the full picture)
- Print `OK    {check-name}` and `FAIL  {check-name}: {diagnostic}` per check (mirrors `scripts/install_smoke.py` print pattern)

Pure Python; pathlib.Path; type hints on all functions; no third-party imports (the `build` package is invoked via subprocess so release_gate.py itself stays import-free of build).
</release_gate_contract>
