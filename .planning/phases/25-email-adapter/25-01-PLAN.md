## EmailAdapter shape

```
class EmailAdapter:
    name = "email"

    def __init__(self) -> None:
        self._task = None
        self._context = None
        self._poll_interval = DEFAULT_POLL_INTERVAL
        self._config = None  # EmailConfig snapshot

    def bind(self, app, context):
        return None

    async def start(self, context):
        self._context = context
        try:
            email_config = _load_email_config_from_env()
        except _MissingEnvError as exc:
            context.registry.mark_error(self.name, str(exc))
            return
        self._config = email_config
        self._poll_interval = email_config.poll_interval
        self._task = asyncio.create_task(self._poll_loop())
        context.registry.mark_running(self.name)

    async def stop(self):
        if self._task is not None:
            self._task.cancel()
            with contextlib.suppress(BaseException):
                await self._task
```

## Poll loop

```
async def _poll_loop(self):
    backoff = self._poll_interval
    while True:
        try:
            await asyncio.to_thread(self._poll_once)
            backoff = self._poll_interval
        except asyncio.CancelledError:
            raise
        except Exception as exc:
            if self._context is not None:
                self._context.registry.mark_error(self.name, f"{type(exc).__name__}: {exc}")
            await asyncio.sleep(backoff)
            backoff = min(backoff * 2, 300)
            continue
        await asyncio.sleep(self._poll_interval)
```

## _poll_once

`_poll_once` is fully synchronous and runs in a worker thread:

1. `IMAP4_SSL(host, port)` -> `login(user, password)` -> `select("INBOX")`
2. `search(None, "UNSEEN")` -> list of UIDs
3. For each UID: `fetch(uid, "(BODY.PEEK[])")` (PEEK = do not flip `\Seen`)
4. Parse raw bytes with `BytesParser(policy=email.policy.default).parsebytes(raw)`
5. If `_should_process(msg)` is False: mark seen and skip
6. Extract plain-text body, strip quoted reply
7. Try `run_agent(...)`; on success build reply, send via SMTP
8. Mark message seen via `store(uid, "+FLAGS", "(\\Seen)")` after SMTP ack
9. If agent raised: mark seen anyway, register error, continue loop
10. `logout()` IMAP and `quit()` SMTP

## Reply construction

```
def _build_reply(original, response_text, from_addr):
    reply = EmailMessage()
    reply["From"] = from_addr
    reply["To"] = original.get("Reply-To") or original.get("From")
    subj = original.get("Subject", "")
    if not subj.lower().startswith("re:"):
        subj = f"Re: {subj}"
    reply["Subject"] = subj
    msg_id = original.get("Message-ID")
    if msg_id:
        reply["In-Reply-To"] = msg_id
        prior = original.get("References", "")
        refs = (prior + " " + msg_id).strip() if prior else msg_id
        reply["References"] = refs
    reply["Message-ID"] = make_msgid()
    reply["Date"] = formatdate(localtime=False, usegmt=True)
    reply.set_content(response_text)
    return reply
```

## Quote stripping

```
_QUOTE_PATTERNS = [
    re.compile(r"^On .+ wrote:\s*$", re.MULTILINE),
    re.compile(r"^Le .+ a ecrit\s*:\s*$", re.MULTILINE),
    re.compile(r"^From:.+", re.MULTILINE),  # Outlook citation header
]

def _strip_quoted_reply(text):
    earliest = len(text)
    for pat in _QUOTE_PATTERNS:
        m = pat.search(text)
        if m and m.start() < earliest:
            earliest = m.start()
    out = text[:earliest]
    # Drop trailing `>` quote lines.
    lines = out.splitlines()
    while lines and lines[-1].startswith(">"):
        lines.pop()
    return "\n".join(lines).strip()
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| `HORUS_OS_EMAIL_*` env vars -> `imaplib.IMAP4_SSL.login` and `smtplib.SMTP_SSL.login` | A leaked app password grants full mailbox access |
| Inbound email body -> `run_agent` | Adversary can prompt-inject the agent via any email sent to the inbox |
| `From` header -> sender-filter check | The `From` header is unauthenticated unless DKIM/SPF/DMARC are enforced server-side; the adapter trusts what IMAP returned |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-25-01 | Information Disclosure | Email password printed in logs or exception messages | mitigate | The adapter never logs the password; exceptions are captured as `type(exc).__name__: str(exc)` which on auth failures produces messages like `IMAP4.error: AUTHENTICATIONFAILED` without echoing the password. The setup guide warns operators to use app passwords and rotate on leak |
| T-25-02 | Spoofing | Adversary spoofs `From` header to bypass the self-filter | accept | The self-filter is a convenience to avoid bot-on-bot loops, not a security boundary. A spoofed `From` matching the SMTP user means the spoofed message is silently dropped, which is the safe behavior. Real spoofing protection lives in SPF/DKIM/DMARC at the receiving server |
| T-25-03 | Tampering | Prompt injection in an email body causes the agent to perform unintended actions | accept | The adapter calls `run_agent(..., tools=None)` for v0.3. Phase 27 / v0.4 will add per-channel tool gates. Documented in the setup guide |
| T-25-04 | Denial | Adversary floods the inbox with messages, exhausting provider quota | accept | v0.3 leaves rate-limiting to the operator; mail providers also apply send/receive throttles. A per-sender cooldown is a v0.4 backlog item |
| T-25-05 | Elevation of Privilege | App password grants more than mail access (e.g., Gmail app passwords grant full account scope) | mitigate | The setup guide tells operators to use a dedicated bot inbox where possible, and reminds them that an app password is not scoped per service. iCloud and Fastmail app passwords are mail-only |
| T-25-06 | Repudiation | No record of what the bot replied via email | accept | The dashboard's trace table captures every `run_agent` invocation; the response text is stored. Phase 27 will surface this in the UI |
| T-25-07 | Denial (poison message) | An inbound message that always crashes `run_agent` blocks the poll loop forever | mitigate | The adapter marks the message seen even on `run_agent` failure to prevent an infinite reprocess loop, at the cost of dropping that message's reply. The error is logged via `registry.mark_error` |
</threat_model>
