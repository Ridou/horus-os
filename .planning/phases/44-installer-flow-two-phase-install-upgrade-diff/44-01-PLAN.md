<objective>
Ship `horus-os plugins {install, uninstall, list, info, enable, disable, update, grant, revoke}` as a new argparse subparser. The `install` subcommand is the highest-stakes path: it wraps `pip` via `subprocess.run([sys.executable, "-m", "pip", ...])` with `--require-virtualenv` and implements a literal five-phase pipeline (Download → Validate → Grant → Install → Verify) with rollback on any post-Phase-D failure. Sdists, wheels containing `.pth` files, and any spec that would downgrade horus-os runtime deps are refused before Phase D ever runs. `update` consumes Phase 43's manifest-hash logic via PermissionService.pending_on_upgrade to surface a re-prompt when the new manifest expands the capability set.

Purpose: This is the user-facing trust boundary for third-party plugins. Pitfall 4 (six concrete failure modes of `pip install`-wrapped installers) and Pitfall 5 (silent capability expansion on upgrade) are both prevented HERE — once a wheel reaches Phase D the design assumes the manifest is valid, the capabilities are granted, and the venv is the right one. Every refusal lands before D so a corrupted venv state is not reachable through the supported CLI path.

Output:
- `src/horus_os/plugins/installer.py` — the five-phase pipeline + upgrade-diff classifier + sha256 freeze round-trip
- `src/horus_os/cli/plugins_cmd.py` — argparse dispatcher with nine subcommands
- `src/horus_os/__main__.py` — top-level `plugins` subparser wired in
- 10 test files in `tests/plugins/` + 4 fixture wheel skeletons in `tests/fixtures/installer/`
</objective>

Raise `PluginInstallError(phase="validate", reason="missing_manifest")` if neither location has it.

    8. **`read_wheel_record(wheel_path: Path) -> list[tuple[str, str, str]]`** — parse the wheel's `*.dist-info/RECORD` file (CSV-ish, fields per row: filename, hash, size). Return the list of triples. The `.pth` check iterates and matches `row[0].endswith(".pth")`.

    9. **`read_wheel_metadata(wheel_path: Path) -> email.message.Message`** — parse `*.dist-info/METADATA` via `email.parser.BytesParser().parse()`. Caller iterates `meta.get_all("Requires-Dist", [])` for the downgrade check.

    10. **`check_no_pth(wheel_path: Path) -> None`** — raises `PluginInstallError(phase="pth", reason="pth_in_record")` if any RECORD entry's filename ends with `.pth`. Message includes the offending filename. Reads RECORD via the helper above.

    11. **`check_no_downgrade(wheel_path: Path, current_freeze: dict[str, str]) -> None`** — parses `Requires-Dist` lines, builds `Requirement` objects via `packaging.requirements.Requirement`, and for each requirement whose `req.name.lower()` is in `HORUS_OS_RUNTIME_DEPS`, checks that `current_freeze[req.name.lower()]` satisfies `req.specifier`. If not, raise `PluginInstallError(phase="downgrade", reason="runtime_dep_downgrade")` whose message names the package, the current version, and the offending specifier.

    12. **`detect_sdist(download_dir: Path) -> bool`** — returns True if no `*.whl` and at least one `*.tar.gz` or `*.zip` is in the dir.

    13. **`render_grant_prompt(spec: PluginSpec, stdout: TextIO) -> None`** — prints the literal scheme:
        ```
        Plugin {spec.name} {spec.version} requests these capabilities:

          [a] filesystem.read — Read files from disk paths the plugin declares. Does NOT include writing, deleting, or modifying files.
          [b] net.outbound    — Open outbound network connections to hosts the plugin declares...

        Grant all (y) / per-capability (a/b/c/...) / refuse (n)?
        ```
        The per-cap letter is computed from `string.ascii_lowercase[i]` for the ith capability. DESCRIPTIONS lookup is `DESCRIPTIONS[Capability(cap.name)]`.

    14. **`prompt_for_grants(spec: PluginSpec, *, stdin: TextIO, stdout: TextIO, assume_yes: bool) -> set[str]`** — returns the set of capability name strings the user granted. When `assume_yes=True` (the `--yes` flag bypass), returns the full set of `cap.name for cap in spec.capabilities` without prompting. Otherwise renders the prompt, reads a line, strips. `y` → full set. `n` → raise `PluginInstallError(phase="grant", reason="user_refused_grant")`. A letter string like `ab` → that subset. Refusing ANY of the requested capabilities while granting others → raise `PluginInstallError(phase="grant", reason="partial_grant_refused")` because INSTALL-05 forbids the half-grant state.

    15. **`install_plugin(spec_str: str, *, db: Database, allow_sdist: bool, allow_system_python: bool, assume_yes: bool, stdin: TextIO, stdout: TextIO, stderr: TextIO) -> str`** — the orchestrator. Returns the installed plugin name on success. Pipeline:

        - **Phase 0 (venv gate):** if `not is_venv() and not allow_system_python`, raise `PluginInstallError(phase="venv", ...)`.

        - **Phase A.5 (sdist gate):** `if detect_sdist(tmpdir) and not allow_sdist`, raise `PluginInstallError(phase="sdist", ...)`. Cleanup tmpdir.
        - **Phase B (validate):** locate the `.whl` in tmpdir (or `.tar.gz` if `allow_sdist`); call `extract_horus_plugin_toml(wheel_path)` → bytes → `validate_manifest(bytes)` → PluginSpec. Catch `ValidationError` and re-raise as `PluginInstallError(phase="validate", reason="manifest_validation_failed")` whose message is `format_validation_error(exc)`. Call `check_no_pth(wheel_path)`. Capture `pre_freeze = pip_freeze_sha256()` and `pre_freeze_parsed = parse_freeze(run_pip("freeze").stdout)`. Call `check_no_downgrade(wheel_path, pre_freeze_parsed)`.
        - **Phase C (grant):** `granted = prompt_for_grants(spec, stdin=stdin, stdout=stdout, assume_yes=assume_yes)`. For each cap in `granted`, call `PermissionService(db).grant(spec.name, spec.version, cap, actor='cli', manifest_hash=spec.manifest_hash)`. (If the user refused, the prompt raises and we never reach this loop, so no half-grant state.)
        - **Phase D (install):** `run_pip("install", "--no-deps", "--no-build-isolation", "--require-virtualenv" if not allow_system_python else "--no-build-isolation", str(wheel_path))`. (If allow_system_python is True we omit --require-virtualenv since pip would refuse outside a venv.)
        - **Phase E (verify):** `post_freeze = pip_freeze_sha256()`. If `post_freeze == pre_freeze`, raise `PluginInstallError(phase="verify", reason="silent_rollback")` and rollback. Compare `parse_freeze(post)` to `parse_freeze(pre)` for HORUS_OS_RUNTIME_DEPS — any mismatch → raise `PluginInstallError(phase="verify", reason="runtime_dep_changed")` and rollback. Call `discover_plugins()` (note: tests may need to patch this to avoid scanning system entry points); assert `spec.name in {s.name for s in specs}`. INSERT the `plugins` row with status='pending' (via direct sqlite or via PluginRegistry.register if the spec is reconstructable — the simplest path is the raw INSERT since the plugin entry-point distribution metadata is the new install).
        - **Rollback:** any exception from Phase D onward triggers `run_pip("uninstall", "-y", spec.name)` AND `DELETE FROM plugins WHERE name = ?`. The exception is then re-raised; the caller (the CLI) translates it into a non-zero exit code with a printed message.

    16. **`uninstall_plugin(name: str, *, db: Database) -> None`** — `run_pip("uninstall", "-y", name)`; `DELETE FROM plugins WHERE name=?`; the plugin_capabilities and plugin_status rows CASCADE on delete via the FK.

    17. **`update_plugin(name: str, spec_str: str, *, db: Database, assume_yes: bool, stdin: TextIO, stdout: TextIO, stderr: TextIO, **kwargs) -> str`** — the upgrade-diff. Read the existing `plugins.manifest_hash` and `plugins.version`. AND plugin_version=? AND state='granted'`). Run Phase D; on success, for each cap in `old - new`, log a 'revoked' audit row via PermissionService.revoke(actor='cli'). <threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| user CLI → pip subprocess | The `horus-os plugins install <spec>` command spawns a `pip` subprocess. The spec string crosses from user input into a `[sys.executable, "-m", "pip", "install", spec]` argv list. Untrusted: a malicious user (or a copy-pasted command from a phishing page) could supply a spec that resolves to a typosquatted PyPI package. |
| pip download → wheel extraction | The downloaded wheel is parsed by `zipfile` and its METADATA + RECORD inspected. Untrusted: the wheel comes from PyPI (or a `git+https` URL) and could contain malicious bytes. |
| pip install → venv | Phase D mutates the active Python venv. Untrusted: a successful install grants the wheel author whatever code-execution capability `setup.py` (for sdist) or the package's `__init__.py` (for wheel) wants to use. |
| grant prompt → SQLite | The user types a grant decision (`y`/`n`/`abc`) that becomes durable state in `plugin_capability_grants_log`. Trusted (local user input) but must not be forgeable: the actor field has a CHECK constraint to `{'cli','dashboard','system'}`. |
| upgrade-diff classifier → re-prompt | A malicious plugin upgrade (Pitfall 5 — author's PyPI account compromised) could expand capabilities silently. The classifier must surface the EXPANSION as a re-prompt; "silently inherit" is a denial of trust. |
