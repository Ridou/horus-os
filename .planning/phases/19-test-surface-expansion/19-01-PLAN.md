# horus-os Phase 19 Plan 01

## Stubbing patterns to reuse

### Pattern 1: provider Conversation stub (multi-agent E2E)

`run_agent_loop` calls `_new_conversation(provider, model, system_prompt=...)`.
Monkeypatching this returns a fake Conversation whose `.send(...)`
returns scripted `AgentResult` values: turn 1 has a `delegate_to_agent`
tool_use; turn 2 (after the delegate handler runs) has text only.

### Pattern 2: anthropic SDK swap (provider streaming)

`tests/test_provider_anthropic.py` swaps the `anthropic` module via
`sys.modules`. A FakeStreamingAnthropic with a `stream` method returns
an async context manager whose `text_stream` is an async generator
that can raise mid-iteration.

### Pattern 3: fake entry_points (adapter discovery)

`tests/test_adapters_integration.py::_stub_entry_points` returns a
canned list of `_FakeEntryPoint(name, target)` for the
`horus_os.adapters` group. The fake adapter class declares a `bind`
method that mounts an HMAC-signed route directly.

### Pattern 4: CLI run with stubbed loop

`tests/test_cli_run.py` calls `main(argv, stdout, stderr)` after
seeding `init`, monkeypatches `run_cmd.run_agent_loop`, and asserts on
captured kwargs.

### Pattern 5: SSE end-to-end

`tests/test_server_stream.py` uses `TestClient(create_app(data_dir))`
plus a `_frames(body)` helper to parse `data: <json>\n\n` frames.

### Pattern 6: v0.1 database fixture

`tests/test_storage.py::test_schema_v1_database_upgrades_to_current`
shows the exact DDL for a v1 traces table. Reuse it verbatim, then
hand the resulting db_path to `create_app(data_dir)` via a `Config`
that points db_path at the legacy file.
