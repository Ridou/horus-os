# horus-os Phase 27 Plan 01

## Existing FastAPI surface (verified)

`create_app(data_dir)` builds a FastAPI app with a lifespan that
calls `start` and `stop` on lifecycle adapters. Status lives in
`app.state.adapter_registry` (`AdapterRegistry`). Existing route:

- `GET /api/adapters` returns a list of `{name, status,
  last_activity_at, error_count, error_message}` entries sorted by
  name.

The discovery + bind block runs at `create_app` time:

```
_adapters = discover_adapters()
_registry = AdapterRegistry()
for _adapter in _adapters:
    _registry.register(_adapter.name)

_adapter_context = AdapterContext(
    config=...,
    data_dir=_resolved_data_dir,
    registry=_registry,
)
```

`AdapterContext` accepts an optional `tool_registry: ToolRegistry
| None = None` (Phase 26). Today nothing in `create_app` passes
it, so tool-providing adapters like Calendar can never register
tools through the dashboard's app. Phase 27 fixes that.

## Lifecycle dispatch pattern

The lifespan uses `hasattr(adapter, "start")` and
`hasattr(adapter, "stop")` for runtime dispatch, capturing
errors into `mark_error`. The new toggle routes use the same
pattern. The discovered adapter list lives in the `create_app`
closure; the routes need access to that list, so we lift it onto
`app.state.adapters` as well (small, derived state, parallel to
`adapter_registry`).

## tool_registry layering

Today `/api/chat` builds its own per-request `ToolRegistry` via
`_build_default_registry`. The Calendar adapter wants to register
its tools onto an app-level registry at bind time. The minimal
move: build one `app_tool_registry = ToolRegistry()` in
`create_app`, pass it through `AdapterContext`, and stash it on
`app.state.tool_registry`. The chat path continues to use its
per-request registry for now; merging them is a v0.4 concern
because the chat path needs to compose default tools with
adapter-provided tools in a way that respects per-profile
`allowed_tools` filtering, which is out of scope here.

The Phase 26 contract that a tool-providing adapter sees a
non-None `tool_registry` on the context is now honored by
`create_app`. That is the test target.

## Route shapes

```
POST /api/adapters/{name}/disable
  200 -> {"name": "...", "status": "stopped"}
  404 -> {"detail": "adapter '...' not found"}
  400 -> {"detail": "adapter '...' does not support disable; it has no stop() hook"}
  500 -> {"detail": "stop hook raised: ...: ..."}

POST /api/adapters/{name}/enable
  200 -> {"name": "...", "status": "running"}
  404 -> {"detail": "adapter '...' not found"}
  400 -> {"detail": "adapter '...' does not support enable; it has no start() hook"}
  500 -> {"detail": "start hook raised: ...: ..."}
```

The 500 path still updates the registry to `error` so the
dashboard's next poll surfaces the failure.

## /api/adapters payload extension

```
{
  "adapters": [
    {
      "name": "webhook",
      "status": "running",
      "last_activity_at": "...",
      "error_count": 0,
      "error_message": null,
      "supports_toggle": false
    },
    ...
  ]
}
```

`supports_toggle` is computed at request time:
`hasattr(adapter, "start") and hasattr(adapter, "stop")`. Adapters
without both hooks render in the dashboard with a disabled n/a
button rather than a confusing 400 on click.

## Dashboard tab shape

New nav button: `<button data-tab="adapters">Adapters</button>`
placed after Writes. New section:

```
<section class="tab" id="adapters">
  <div class="row">
    <h2 style="margin: 0; font-size: 14px;">Adapter status</h2>
    <span class="grow"></span>
    <button class="ghost" id="refresh-adapters">Refresh</button>
  </div>
  <div id="adapters-body"><div class="empty">loading</div></div>
</section>
```

The JS adds:

- `loadAdapters()` async function: fetch `/api/adapters`, render a
  table. Each row has the name, a status pill, last_activity_at,
  error_count, error_message (truncated to ~80 chars; full text in
  the `title` attribute), and a toggle button. The button calls
  `toggleAdapter(name, action)` where `action` is `enable` or
  `disable`.
- `toggleAdapter(name, action)` async function: POST
  `/api/adapters/{name}/{action}` with no body. On success, call
  `loadAdapters()` to refresh. On error, alert the message.
- New status pill style `.pill.muted` for `stopped` status (gray
  using `var(--text-muted)`).
- A polling interval started when the tab activates, cleared when
  the tab deactivates. Implemented by extending the existing
  `setTab(name)` logic with a `clearInterval(adaptersPoll)` and a
  fresh `setInterval(loadAdapters, 5000)` when switching to
  adapters.

3. Extend the existing `list_adapters` handler to derive

   `_registry.entries()`; we need to cross-reference each entry
   with the matching adapter from `_adapters` to read its hook

```
@app.get("/api/adapters")
def list_adapters() -> dict[str, Any]:
    """Return the per-adapter status snapshot from the registry."""
    by_name = {a.name: a for a in _adapters}
    out: list[dict[str, Any]] = []
    for entry in _registry.entries():
        adapter = by_name.get(entry.name)
        supports_toggle = (
            adapter is not None
            and hasattr(adapter, "start")
            and hasattr(adapter, "stop")
        )
        out.append(
            {
                "name": entry.name,
                "status": entry.status,
                "last_activity_at": entry.last_activity_at,
                "error_count": entry.error_count,
                "error_message": entry.error_message,
                "supports_toggle": supports_toggle,
            }
        )
    return {"adapters": out}
```

4. Add the two new routes immediately after `list_adapters`:

```
@app.post("/api/adapters/{name}/disable")
async def disable_adapter(name: str) -> dict[str, Any]:
    by_name = {a.name: a for a in _adapters}
    adapter = by_name.get(name)
    if adapter is None:
        raise HTTPException(404, detail=f"adapter {name!r} not found")
    stop = getattr(adapter, "stop", None)
    if stop is None:
        raise HTTPException(
            400,
            detail=f"adapter {name!r} does not support disable; it has no stop() hook",
        )
    try:
        await stop()
    except Exception as exc:
        _registry.mark_error(name, f"{type(exc).__name__}: {exc}")
        raise HTTPException(500, detail=f"stop hook raised: {type(exc).__name__}: {exc}") from exc
    _registry.mark_stopped(name)
    return {"name": name, "status": _registry.get(name).status}


@app.post("/api/adapters/{name}/enable")
async def enable_adapter(name: str) -> dict[str, Any]:
    by_name = {a.name: a for a in _adapters}
    adapter = by_name.get(name)
    if adapter is None:
        raise HTTPException(404, detail=f"adapter {name!r} not found")
    start = getattr(adapter, "start", None)
    if start is None:
        raise HTTPException(
            400,
            detail=f"adapter {name!r} does not support enable; it has no start() hook",
        )
    try:
        await start(_adapter_context)
    except Exception as exc:
        _registry.mark_error(name, f"{type(exc).__name__}: {exc}")
        raise HTTPException(500, detail=f"start hook raised: {type(exc).__name__}: {exc}") from exc
    _registry.mark_running(name)
    return {"name": name, "status": _registry.get(name).status}
```

```
async function loadAdapters() {
  const body = $("#adapters-body");
  try {
    const data = await fetchJson("/api/adapters");
    if (!data.adapters.length) {
      body.innerHTML = '<div class="empty">no adapters discovered</div>';
      return;
    }
    const rows = data.adapters.map((a) => {
      let pillClass = "muted";
      if (a.status === "running") pillClass = "success";
      else if (a.status === "error") pillClass = "error";
      const pill = `<span class="pill ${pillClass}">${escapeHtml(a.status)}</span>`;
      const last = a.last_activity_at || "(never)";
      const errMsg = a.error_message ? escapeHtml(a.error_message.slice(0, 80)) : "";
      let toggle;
      if (!a.supports_toggle) {
        toggle = `<button class="ghost" disabled title="adapter has no start/stop hooks">n/a</button>`;
      } else if (a.status === "running") {
        toggle = `<button class="ghost" data-action="disable" data-name="${escapeHtml(a.name)}">Disable</button>`;
      } else {
        toggle = `<button class="ghost" data-action="enable" data-name="${escapeHtml(a.name)}">Enable</button>`;
      }
      return `<tr><td>${escapeHtml(a.name)}</td><td>${pill}</td><td class="muted">${escapeHtml(last)}</td><td>${a.error_count}</td><td class="muted" title="${escapeHtml(a.error_message || "")}">${errMsg}</td><td>${toggle}</td></tr>`;
    }).join("");
    body.innerHTML = `<table><thead><tr><th>name</th><th>status</th><th>last_activity_at</th><th>errors</th><th>error_message</th><th></th></tr></thead><tbody>${rows}</tbody></table>`;
    body.querySelectorAll("button[data-action]").forEach((btn) => {
      btn.addEventListener("click", () => toggleAdapter(btn.dataset.name, btn.dataset.action));
    });
  } catch (e) {
    body.innerHTML = `<div class="empty">${escapeHtml(e.message)}</div>`;
  }
}

async function toggleAdapter(name, action) {
  try {
    await fetchJson(`/api/adapters/${encodeURIComponent(name)}/${action}`, { method: "POST" });
    loadAdapters();
  } catch (e) {
    alert(e.message);
  }
}

$("#refresh-adapters").addEventListener("click", loadAdapters);
```

  The `data-name` attribute escapes through `escapeHtml` because
  the name comes from the API and we control it, but defense in
  depth is cheap.
