From src/horus_os/adapters/discord_adapter.py (CLOSEST analog for lazy-import + start/stop lifecycle; MIRROR THIS shape):

```
async def start(self, context: AdapterContext) -> None:
    self._context = context
    try:
        import discord
    except ImportError:
        context.registry.mark_error(
            self.name,
            "discord.py is not installed; pip install 'horus-os[discord]'",
        )
        return
    ...
    context.registry.mark_running(self.name)

async def stop(self) -> None:
    if self._client is not None:
        with contextlib.suppress(Exception):
            await self._client.close()
    ...
```

Pattern: lazy-import inside `start()`; on ImportError, registry.mark_error returns WITHOUT raising so siblings keep running. Stop is idempotent (None-checks every handle).

DIFFERENCE FOR OTEL: per Pitfall 12, `start()` must NOT silently mark_error and return; it MUST raise `RuntimeError("OTel adapter requires 'pip install horus-os[otel]'")`. Reason: the OTel adapter is opt-in (user installed `[otel]` extra); if their start fails it is a configuration error they want to see loudly, not a quiet `error` pill in the dashboard. The other adapters (discord etc.) are always-loaded via entry points and silent registry.mark_error is the right posture there; OtelAdapter is different.

The wrap: `context.registry.mark_error(self.name, str(exc))` BEFORE re-raising the RuntimeError so the dashboard ALSO sees the error pill; FastAPI lifespan in server/api.py:100-102 catches the exception, marks the registry, and continues siblings — so the raise does NOT abort startup.
