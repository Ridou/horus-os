# horus-os Phase 20 Plan 01

## What the smoke script currently does (v0.1)

`scripts/install_smoke.py` exposes `main() -> int` and runs eight checks:

1. `--version`
2. `--help` lists subcommands
3. `init --data-dir <tmp>` populates the data dir
4. `init` refuses overwrite without `--force`
5. `init --force` reinitializes
6. `traces --data-dir <tmp>` on empty DB
7. `run "hello"` without API keys exits 2 with the expected error
8. `serve --help` prints flags

All checks share one `tempfile.mkdtemp` data dir and clean it up in `finally`.

## What v0.2 needs added

Inserted in the same `try:` block, after check 5 (init succeeded) and
before check 7 (run without API key still works the same):

- **Schema v4 on disk**: open `<tmp>/horus.sqlite` with stdlib
  `sqlite3`, read `PRAGMA user_version`, assert == 4. List columns on
  `traces`, assert `parent_trace_id` and `agent_profile_name` are present.
  Count rows in `agent_profiles`, assert >= 1. Read the `name` of one
  row, assert == "default".

- **agents list**: `horus-os agents list --data-dir <tmp>` exit 0, stdout
  contains "default".

- **agents CRUD round-trip**: `agents create --name smoke_test
  --system-prompt 'smoke' --data-dir <tmp>` exit 0, stdout contains
  "Created"; `agents show smoke_test --data-dir <tmp>` exit 0, stdout
  contains "smoke_test" and "smoke"; `agents delete smoke_test
  --data-dir <tmp>` exit 0, stdout contains "Deleted".

- **run --agent --no-stream without keys**: same error path as the
  vanilla `run` check but goes through the buffered branch with the
  default profile. Exit 2, stderr mentions an API key.

- **Imports**: a Python -c subprocess that imports
  `run_agent_stream`, `ToolCallEvent`, `Adapter`, `AdapterContext`,
  `discover_adapters` from `horus_os`. Exit 0 confirms the public
  surface is intact on a fresh install.

- **discover_adapters() runs**: same Python -c subprocess calls
  `discover_adapters()` and asserts the returned object is a list. The
  reference webhook adapter is discovered but not bound (no secret
  set; expected).

## Pytest wrapper

`tests/test_install_smoke.py` runs the smoke script under pytest with
`tmp_path`. Two strategies considered:

- (A) Import `scripts.install_smoke` and call `main()` directly. Problem:
  the script uses a fresh `tempfile.mkdtemp` and the CLI binary `horus-os`,
  not `sys.executable -m horus_os`. The CLI binary is present in the
  installed env, so this is fine for the lint+test matrix (where the
  package is installed in editable mode).

- (B) Subprocess the script with `sys.executable scripts/install_smoke.py`.

Strategy (B) wins because it exactly mirrors what CI runs and isolates
the env. The test verifies exit code 0 and a few stdout substrings.

Cross-OS: the test uses `pathlib.Path(__file__).parent.parent /
"scripts" / "install_smoke.py"` to find the script. It does not bake in
any forward slashes. Skip the test when `horus-os` is not on PATH
(e.g. running pytest from a non-installed checkout).

## CI workflow

The current install-smoke job runs `python scripts/install_smoke.py`
with no arguments. The extended script keeps the same no-args call
signature (it still mkdtemps internally). No workflow change required.

If a future iteration wants the data dir injected externally
(e.g. a persistent dir for inspection), the script accepts a
`HORUS_OS_SMOKE_DATA_DIR` env override but defaults to mkdtemp.

## Stubbing patterns to reuse

### Pattern 1: capture_output subprocess

Identical to the Phase 10 `run()` helper. No change needed.

### Pattern 2: stdlib sqlite3 inspection

`conn = sqlite3.connect(str(db_path)); conn.execute('PRAGMA user_version').fetchone()`
is enough. Use a context manager so the connection closes deterministically;
Windows is sensitive to lingering file handles in tempdir teardown.

### Pattern 3: Python -c subprocess for import smoke

`subprocess.run([sys.executable, '-c', PY], ...)` runs in the same env
that ran `pip install '.[all]'`. The horus-os module is importable.
