<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| model output -> shell args | The model (possibly steered by prompt injection from a fetched web page, a note, or user input) produces command + args that reach a real OS subprocess |
| shell process -> filesystem | A spawned process can read and write paths; the working-directory boundary is the only in-process containment |
| package install (none) | This plan adds NO new dependency; shell uses stdlib asyncio.subprocess only, so there is no supply-chain surface in 75-01 |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-75-01 | Elevation of Privilege | shell_exec handler | mitigate | shell=False + structured args list; reject_metacharacters on [command, *args] before any spawn (SE-2); literal shell=True asserted absent by grep gate |
| T-75-02 | Tampering | working-directory boundary | mitigate | resolve_within uses Path.resolve().is_relative_to(); absolute and parent-traversal paths rejected; cwd pinned to the resolved safe root (SE-3) |
| T-75-03 | Denial of Service | subprocess lifetime + output | mitigate | asyncio.wait_for timeout (default 30s) then terminate-then-kill; stdout truncated to 1 MB cap; exit_code None recorded on timeout (SE-4) |
| T-75-04 | Repudiation | audit trail | mitigate | every run writes one ShellInvocation row (command, exit_code, truncated stdout, working_directory) on success, rejection, and timeout (SHELL-02) |
| T-75-05 | Information Disclosure | ShellInvocation.stdout_truncated | accept | truncated stdout is stored verbatim for review per the "inspect everything" value; the audit row is local-only SQLite, same trust level as note_writes content |
| T-75-06 | Tampering | cross-OS subprocess differences | mitigate | shell_type config (bash/powershell/cmd/auto) instead of hardcoded bash; cross-platform terminate()+kill() instead of os.killpg (SE-4); end-to-end echo test on the 3-OS matrix is in plan 75-02 |
| T-75-SC | Tampering | dependency installs | accept | 75-01 adds zero packages (stdlib asyncio.subprocess only); no Package Legitimacy Gate is triggered; the broader v0.8 extras land in Phase 76 |
</threat_model>
