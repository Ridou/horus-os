<objective>
Wire the plan 75-01 shell factory into the agent runtime behind the SHELL-01 double gate, surface the shell state through horus-os doctor, and ship the TEST-36 absence proof plus the cross-OS end-to-end regression test.

The double gate is the whole point: the tool registers only when BOTH the HORUS_OS_SHELL_ENABLED=true env var is set AND the invoking agent profile lists shell_exec in its allowed_tools. Either gate removed removes the capability. A fresh install has neither, so a default agent cannot reach the shell under any prompt.

Purpose: satisfy the BLOCKING SHELL-01 constraint and prove it with a regression test that fails if the gate ever regresses.
Output: gated registration at both registry-build sites, doctor display, double-gate + cross-OS + doctor tests.
</objective>
