<objective>
Build the native Deep Research engine on top of the EXISTING delegation runtime: a session-scoped source registry, a seeded research agent team, a citation-validating report builder, and a ResearchOrchestrator that configures `run_agent_loop` (it does NOT build a new loop) with a hard source-and-iteration budget. This plan delivers the pure-Python core with no API or UI surface yet.

Purpose: This is the flagship of v0.8. It must compose the shipped primitives (`IterationBudget`, `make_delegate_tool`, `run_agent_loop`, `NotesStore`) into an autonomous research capability with three non-negotiable guarantees: hard budget enforcement (RESEARCH-04), source de-duplication by URL (RESEARCH-03), and zero hallucinated citations (every cited URL was actually fetched). The phase is built native to avoid the Alibaba-NLP/DeepResearch port-license entanglement flagged in SEED-004.

Output: `src/horus_os/research/` package (registry, profiles, report, orchestrator), new `research_max_sources` / `research_max_iterations` config fields, and the TEST-37 regression suite proving budget enforcement and de-duplication.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| LLM synthesis output -> rendered report | The model can emit citation URLs it never fetched (training-memory hallucination). |
| Sub-agent -> delegation tool | A sub-agent that could delegate could recursively re-enter the research coordinator. |
| Research run -> token/cost spend | An unbounded loop or repeated identical queries can burn unbounded tokens. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-01 | Information disclosure (false provenance) | report.py citation rendering (DR-2) | mitigate | ReportBuilder validates every citation URL against SourceRegistry.contains before emitting; unverified URLs are rejected via UnverifiedCitation or flagged, never rendered as a clean reference. Test asserts an unregistered URL cannot appear in the reference list. |
| T-73-02 | Elevation of privilege (recursion) | profiles.py + orchestrator.py (DR-3) | mitigate | Every research sub-agent profile sets an explicit non-None allowed_tools that omits delegate_to_agent; orchestrator filters sub-agent registries through _filter_registry with those lists. Test asserts no sub-agent registry contains delegate_to_agent. |
| T-73-03 | Denial of service (runaway cost) | orchestrator.py budget (DR-1) | mitigate | Shared IterationBudget(research_max_iterations) caps the whole delegation tree; SourceRegistry(max_sources) caps fetched sources and raises SourceBudgetExceeded rather than overrunning; both caps are config-driven hard limits proven by TEST-37. |
</threat_model>
