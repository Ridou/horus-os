<objective>
Expose the Phase 73-01 research engine over HTTP: start a run, show the plan before execution with a cancel control, stream live progress, persist the finished report as an audited note, and make the generating trace inspectable. This is the server half of RESEARCH-02 (plan-before-execute + cancel + live progress) and RESEARCH-05 (audited note + inspectable trace + visible task status).

Purpose: RESEARCH-02 requires the plan to be shown and cancelable before any tokens are spent, plus a live progress panel. RESEARCH-05 requires the report to land in the existing audit trail (a reviewable note) and the run's trace to be inspectable in /traces and its status visible in /tasks. The architecture reuses the existing `tasks` table (no schema bump), the existing `NotesStore` audited write path, and the existing FastAPI app factory and loopback-guard idioms.

Output: `src/horus_os/research/api.py` (the research APIRouter), wiring in `server/api.py`'s `create_app`, and the regression suite proving plan-before-execute, cancel, audited-note persistence, and trace linkage.

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Remote HTTP client -> research start/cancel | A non-local client could start or cancel costly research runs on the local server. |
| Background run -> audited notes folder | The report write path must not escape the notes directory. |
| Long-running run -> server resources | A run with no cancel path can spend tokens and hold resources indefinitely. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-73-04 | Elevation of privilege (remote control) | api.py POST /api/research, POST /api/research/{id}/cancel | mitigate | Mutating research routes reuse dashboard_api._require_loopback, identical to DELETE /api/tasks; non-loopback clients get 403. Test asserts the refusal. |
| T-73-05 | Tampering (path escape on report write) | api.py report persistence | mitigate | Report is written via NotesStore.create_note with a fixed research/<task_id>.md rel-path; NotesStore._resolve refuses any path resolving outside notes_dir. No user-controlled path component beyond the server-generated task_id. |
| T-73-06 | Denial of service (uncancelable run) | api.py cancel + orchestrator should_cancel | mitigate | POST cancel flips task status to cancelled and sets a per-task cancel flag the run checks between iterations, halting before the next delegation turn; combined with 73-01's IterationBudget hard cap. |
</threat_model>
