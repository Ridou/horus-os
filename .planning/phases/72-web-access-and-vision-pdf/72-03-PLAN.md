<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| browser/client -> POST /api/uploads | Untrusted file bytes, content type, and filename cross into the local filesystem |
| agent -> Playwright MCP browse tool -> arbitrary URL | A prompt-injected instruction can drive the browser to an internal or metadata URL |
| mcp.toml -> registered MCP server | Only servers explicitly listed gain tool access (phase-71 trust gate) |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-72-10 | Tampering | upload filename path traversal | mitigate | Stored filename is uuid4 + a content-type-derived extension, never the client filename; the resolved path is asserted under the uploads dir |
| T-72-11 | Denial of service | upload size | mitigate | The route caps body size (default 10 MB) and returns 413 without persisting oversized files |
| T-72-12 | Elevation of privilege | arbitrary file type upload | mitigate | content_type allowlist (image/png, image/jpeg, image/gif, image/webp, application/pdf); everything else is 400 |
| T-72-13 | Information disclosure | Playwright browse to internal/metadata URL | mitigate | Browsing inherits the SSRF posture (documented) and the phase-71 MCP trust gate; only explicitly configured servers register; doc cross-references the 169.254.169.254 refusal |
| T-72-14 | Spoofing | XSS via uploaded filename in the dashboard | mitigate | The attached filename is rendered via textContent, never innerHTML |
| T-72-SC | Tampering | @playwright/mcp server (external) | accept | The Playwright MCP server is a user-installed external process gated by the phase-71 mcp.toml allowlist; horus-os adds no Python browser dependency |
</threat_model>
