<objective>
Ship the VIS-01 image analysis and VIS-02 PDF reading tool. A single analyze_file tool factory accepts a local file path, routes images into a provider-native vision block (base64; zero new Python deps for the image path) and PDFs through pypdf text extraction behind a pre-flight size check. All extracted content is wrapped in an UNTRUSTED DOCUMENT CONTENT provenance block to defeat invisible-text prompt injection (Pitfall VN-1). The Anthropic and Gemini Conversation builders are extended to accept multimodal user content so the image path works through the existing provider layer.

Purpose: Both shipped providers are natively multimodal but horus-os has no way to hand the agent an image or PDF. This closes that gap on-thesis (extends the tool/agent runtime, no new app).
Output: tools/vision.py and multimodal message construction in both providers, plus regression tests for the size check, provenance wrap, and multimodal message shape. The [pdf] pyproject extra and the chat-path registration are owned by sibling plans (72-01 owns the pyproject extras block, 72-03 owns the uploads route and registration), so this plan does not touch pyproject.toml or server/api.py and stays conflict-free with the other wave-1 plan.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| uploaded file -> extracted content -> model | A PDF or image can carry invisible text or hidden instructions that reach the model |
| user-supplied path -> filesystem read | A path argument can attempt to escape the intended directory |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-72-06 | Tampering | extracted PDF/image content | mitigate | All extracted content is wrapped in an "UNTRUSTED DOCUMENT CONTENT" provenance block; the model is instructed not to treat document text as instructions (Pitfall VN-1) |
| T-72-07 | Denial of service | PDF size | mitigate | Pre-flight st_size check refuses files over max_bytes (default 10 MB) before any read, bounding memory and extraction cost (VIS-02) |
| T-72-08 | Information disclosure | path argument | mitigate | analyze_file resolves the path under base_dir with Path.resolve() and refuses escapes, mirroring read_file_tool |
| T-72-09 | Repudiation | what the model received | mitigate | A sha256 of the file bytes plus the path is recorded in the returned result so the user can audit the analyzed content |
| T-72-SC | Tampering | pypdf dependency (added by 72-01) | accept | pypdf>=6.12.2 is a known pure-Python PyPI package (BSD-3-Clause); no native build chain, no new untrusted transitive surface |
</threat_model>
