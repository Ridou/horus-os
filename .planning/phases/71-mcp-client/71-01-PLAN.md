<objective>
Build the standalone MCP client core for horus-os: a new src/horus_os/mcp_client/
package that connects to explicitly-configured MCP servers over stdio, SSE, and
streamable-HTTP, discovers their tools via tools/list, sanitizes each tool
description, and registers each discovered tool into the existing ToolRegistry
under a mcp:{server}:{tool} namespace so the agent loop traces them like any
builtin. This plan delivers the four security-critical behaviors as testable units:
the opt-in trust gate (MCP-03 BLOCKING), namespacing with collision refusal (MCP-02
and MC-4), description sanitization against tool poisoning (MC-2), and explicit
cross-OS subprocess teardown (MCP-04 and MC-3). Wiring into the FastAPI lifespan,
the agent/CLI surfaces, and the full cross-OS integration test suite land in 71-02.

Purpose: MCP is the industry-standard tool protocol; one client turns every MCP
server into a horus-os tool for free without forking the core. The trust model is
the first design decision (Pitfall MC-1 is BLOCKING), so this plan implements
namespacing, collision refusal, opt-in config, and sanitization before any wiring.

Output: a new optional [mcp] extra (mcp>=1.27), the mcp_client/ package
(config, sanitize, client, registry), a register_namespaced plus CollisionError
addition to ToolRegistry, and a tests/mcp/ suite covering the unit slice of
TEST-34 (namespacing, opt-in trust, sanitization, teardown call sequence).
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| external MCP server to horus-os | An MCP server (third-party subprocess or remote HTTP endpoint) returns tool names, tool descriptions, and tool-call results that are UNTRUSTED. They cross into the agent tool registry and ultimately into the model context. |
| config file to runtime | The mcp.toml allowlist is the ONLY activation surface. Anything not listed there must gain zero tool access (MCP-03 BLOCKING). |
| stdio subprocess lifecycle | A spawned MCP server process holds OS resources (handles, memory) that must be reclaimed on shutdown across macOS, Ubuntu, and Windows (MCP-04). |
