<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| mcp.toml allowlist to running app | At boot, create_app reads ONLY the explicit mcp.toml. An absent file means zero MCP servers and zero MCP tools (MCP-03 BLOCKING). |
| MCP server subprocess to OS | stdio servers are real child processes whose handles and memory must be reclaimed at shutdown on every supported OS, especially Windows where the SDK lifespan cleanup is broken (MCP-04). |
| MCP tool result to trace store | Tool-call results flow into the existing tool_invocations capture path; they must not be allowed to bypass tracing (ARCHITECTURE.md Anti-Pattern 2). |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-71-06 | Spoofing | boot-time server activation (server/api.py) | mitigate | MCP-03 at integration level: create_app loads only mcp.toml; no file means no MCP tools registered. Pinned by test_boot_without_mcp_toml_registers_no_mcp_tools. |
| T-71-07 | Elevation of Privilege | builtin shadowing across two configured servers (server/api.py) | mitigate | MCP-02: builtin_names is computed AFTER the builtin and plugin baseline is registered; register_namespaced refuses a server advertising read_file; the CollisionError surfaces on MCPRegistry.errors() and does not abort the other server. Pinned by test_two_servers_same_toolname_both_namespaced. |
| T-71-08 | Denial of Service (zombie processes) | stdio subprocess teardown (client.py via lifespan) | mitigate | MCP-04 / MC-3 / TEST-34: a real stdio subprocess is spawned and proven dead after stop() on macOS, Ubuntu, and Windows with no Windows skip. Pinned by test_stdio_subprocess_dead_after_stop. |
| T-71-09 | Repudiation / Tampering | MCP tool invocation bypassing tracing (loop.py) | mitigate | MCP-01: MCP tools are ordinary ToolRegistry entries, so execute_tool_uses publishes a tool_invocations event for each call. Pinned by test_mcp_tool_call_writes_trace_row. |
| T-71-SC | Tampering | mcp package install (pip) | mitigate | mcp>=1.27 verified legitimate in 71-01 (PyPI mcp 1.27.2, MIT, Anthropic official, STACK.md). Opt-in [mcp] extra; no new install task in this plan. |
