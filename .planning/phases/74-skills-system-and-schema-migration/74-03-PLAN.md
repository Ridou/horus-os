Make skills usable by agents and the user. Add the `use_skill(name)` tool with progressive disclosure (only names + descriptions injected at turn start; full body loaded on call), enforce the three Odysseus-derived skill pitfalls (SK-1 code-bearing skills require a capability grant, SK-2 skill/plugin/memory boundary, SK-3 skill sub-tool calls honor the invoking agent's allowed_tools with no trusted bypass), and ship the `horus-os skills list` / `horus-os skills show <name>` CLI plus the `horus-os doctor` skills-folder report.

The agent must never gain a trusted execution context that bypasses the registry (PITFALLS SK-3).

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| skill body -> agent execution | A skill body is untrusted instruction text (ToxicSkills attack class). Executing it must not widen the agent's tool reach or run embedded code without a grant. |
| code-bearing skill -> capability system | A `kind: code` skill is the high-risk class; it must reuse the v0.5 default-deny capability grant, never an implicit trust. |
| skill name -> tool/plugin namespace | A skill could shadow a builtin or plugin tool name to hijack a call. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-08 | Elevation of privilege | code-bearing skill execution (SK-1) | mitigate | SkillExecutor refuses to run a `kind: code` skill unless SKILL_EXEC is in the invoking profile's granted capabilities; default-deny (empty grant set) blocks code skills; regression test asserts the refusal. |
| T-74-09 | Elevation of privilege | skill sub-tool call bypassing allowed_tools (SK-3) | mitigate | Sub-tool calls dispatch through `_filter_registry(master, effective_allowed_tools)`; a tool outside the agent's allowed_tools is absent and raises KeyError; no direct handler call; regression test proves the authorization error. |
| T-74-10 | Spoofing | skill shadowing a builtin/plugin tool (SK-2) | mitigate | Registration refuses a skill whose name collides with a registered tool or discovered plugin; a colliding skill cannot shadow a real tool; regression test asserts the refusal. |
| T-74-11 | Information disclosure | full skill body injected into every system prompt | mitigate | Progressive disclosure: only list_skill_summaries() (name + description) enters the level-1 menu; the body loads on use_skill; the e2e test asserts the body is absent from the menu. |
| T-74-12 | Denial of service | skill sub-loop runaway recursion | mitigate | The skill sub-loop reuses the shared IterationBudget so a skill cannot escape the run-level budget (mirrors delegation safety valve). |
</threat_model>
