<objective>
Build the skills subsystem foundation: a Skill dataclass, a filesystem-backed SkillStore that discovers `skills/<name>.md` files with YAML frontmatter (name, description, tags) and a Markdown body, the config wiring for a `skills_dir`, and the `horus-os init` seeding of an example skill. This plan stops before the agent integration; it delivers the store and the discovery contract that plans 74-02 (migration mirror) and 74-03 (use_skill tool + CLI) consume.

Purpose: SKILL-01 requires a skill to be a discoverable file. The filesystem-backed store mirrors the existing NotesStore pattern (same philosophy as memory notes per FEATURES.md) and keeps skills distinct from notes (a separate folder) and from plugins (no Python imports here).

Output: `src/horus_os/skills/` package (types + store), `config.skills_dir`, init seeding, and the regression tests proving discovery, frontmatter parse, and path-escape defense.
</objective>

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| skill file on disk -> SkillStore | A skill file is untrusted text (may come from a shared repo per ToxicSkills). Frontmatter and body are parsed but NOT executed in this plan. |
| skill rel_path -> filesystem | A crafted skill name or symlink could attempt to read outside skills_dir. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-74-01 | Tampering | parse_skill_markdown YAML/frontmatter | mitigate | Hand-rolled restricted key:value parser; reject non-mapping frontmatter with SkillParseError; never eval, never import a YAML loader that can construct arbitrary objects. |
| T-74-02 | Information disclosure | SkillStore._resolve path traversal | mitigate | Mirror NotesStore path-escape defense; resolved path must stay under skills_dir or raise PermissionError (covered by a regression test). |
| T-74-03 | Denial of service | discovery on a broken skill file | mitigate | list_skills() skips unparseable files into list_skill_errors() instead of raising; init/startup never crashes on one bad skill (mirrors plugin DiscoveryError side-channel). |
| T-74-04 | Elevation of privilege | code-bearing skill body | accept (deferred to 74-03) | This plan parses kind="code" but executes nothing; SK-1 enforcement (capability grant for code skills) lands in 74-03 before any body is acted on. No code path in this plan runs a skill body. |
</threat_model>
