<sqlite_persister_canonical>
SQLitePersister design, locked. Implement in src/horus_os/observability/persist.py.

Constructor:
- __init__(self, db: horus_os.storage.Database) -> None

Public entry point:
- on_event(self, event: ObservationEvent) -> None
  Routes by event.kind. LLM_CALL → _insert_llm_call. TOOL_CALL → _insert_tool_invocation. RUN_END → _rollup_trace.

Hard assertion (Pitfall 3 guard):
- At the top of _insert_llm_call and _insert_tool_invocation: `assert event.latency_ms >= 0, f"negative latency_ms={event.latency_ms} on {event.kind} for trace {event.trace_id}"`. Refuse to insert; let the assertion fail the test.

_insert_llm_call: opens a sqlite3 connection via the same path/pragmas as Database._connect (NOT by calling Database._connect directly; _connect is private; create a sibling helper or pass the path and replicate the four PRAGMA calls). INSERTs into llm_calls all 17 columns from the LLMCallEvent.

_insert_tool_invocation: same pattern for tool_invocations table, all 12 columns from the ToolCallEvent.

_rollup_trace: on RUN_END, runs ONE UPDATE on traces using SUM(...) subqueries from llm_calls keyed by trace_id, plus the RunEndEvent.latency_ms as total_duration_ms:
```
UPDATE traces
SET total_input_tokens  = (SELECT COALESCE(SUM(input_tokens),  0) FROM llm_calls WHERE trace_id = ?),
    total_output_tokens = (SELECT COALESCE(SUM(output_tokens), 0) FROM llm_calls WHERE trace_id = ?),
    total_cost_usd      = (SELECT SUM(cost_usd) FROM llm_calls WHERE trace_id = ?),  -- intentionally NULL-preserving; pricing missing rows have cost_usd NULL
    total_duration_ms   = ?
WHERE trace_id = ?
```
(The triple-backtick block above is canonical-SQL prose, not implementation code; transcribe into the actual call site verbatim.)

Note: total_cost_usd intentionally does NOT use COALESCE; if every llm_calls row for the trace has cost_usd NULL (unknown model), the rollup stays NULL (Pitfall 5: NULL is honest, zero is a lie). For input/output tokens we use COALESCE because tokens are always known.

Phase 32 behaviour when no traces row exists for the trace_id: silently no-op (the UPDATE matches 0 rows). Phase 33 wires the runner and guarantees the traces row is INSERTed before RUN_END fires; this phase's tests publish events for trace_ids that DO have a traces row (created via Database.record_trace first).
</sqlite_persister_canonical>
