<objective>
Ship the Google Calendar adapter. Unlike the Discord, Slack,
and Email adapters that ingest events into `run_agent`, the
Calendar adapter exposes tools the agent can call:
`list_calendar_events_today` (always when bind succeeds) and
`create_calendar_event` (only when
`HORUS_OS_CALENDAR_WRITE_ALLOWED == "true"`).

CAL-01 (list tool exposed, structured event response) and
CAL-02 (create tool gated by env var) both land in this single
plan. The adapter implements only `Adapter` (no
`LifecycleAdapter`) and does all its work in `bind`. The Google
libraries (`google-api-python-client`, `google-auth-oauthlib`)
are optional extras and imported lazily.

This plan also extends `AdapterContext` with an optional
`tool_registry` field, the minimum scope expansion needed for a
tool-providing adapter to register tools at bind time. All
existing callers continue to work because the field has a None
default.

extension, an optional pyproject extra and entry-point, one new
test file with fake google modules, and
`docs/adapters/CALENDAR.md`. All 406 baseline tests still pass.

## AdapterContext extension

`src/horus_os/adapters/base.py` gains one optional field:

```
from horus_os.tools.registry import ToolRegistry

@dataclass(frozen=True)
class AdapterContext:
    config: Config
    data_dir: Path
    registry: AdapterRegistry = field(default_factory=AdapterRegistry)
    tool_registry: ToolRegistry | None = None
```

The import is conditional-safe (`ToolRegistry` already ships
in v0.1). Existing tests construct `AdapterContext(config=...,
data_dir=...)` with positional or named args; they continue
to compile because `tool_registry` defaults to None.

## CalendarAdapter shape

```
class CalendarAdapter:
    name = "calendar"

    def __init__(self) -> None:
        self._creds: Any = None
        self._context: AdapterContext | None = None

    def describe(self) -> dict[str, Any]:
        return {
            "name": self.name,
            "version": 1,
            "transport": "google-calendar-api",
            "auth": "oauth2-installed-app",
            "env": OAUTH_CLIENT_PATH_ENV,
        }

    def bind(self, app: Any, context: AdapterContext) -> None:
        context.registry.register(self.name)
        self._context = context

        if context.tool_registry is None:
            context.registry.mark_error(
                self.name,
                "AdapterContext.tool_registry is None; CalendarAdapter needs one to register tools",
            )
            return

        try:
            from google.auth.transport.requests import Request
            from google.oauth2.credentials import Credentials
        except ImportError:
            context.registry.mark_error(
                self.name,
                "google-api-python-client / google-auth-oauthlib are not installed; pip install 'horus-os[calendar]'",
            )
            return

        client_path = os.environ.get(OAUTH_CLIENT_PATH_ENV)
        if not client_path:
            context.registry.mark_error(
                self.name,
                f"{OAUTH_CLIENT_PATH_ENV} is not set; download the OAuth client JSON from Google Cloud and point this env var at it",
            )
            return

        token_path = context.data_dir / TOKEN_FILENAME
        if not token_path.exists():
            context.registry.mark_error(
                self.name,
                f"token file not found at {token_path}; run the OAuth bootstrap (see docs/adapters/CALENDAR.md)",
            )
            return

        try:
            creds = Credentials.from_authorized_user_file(str(token_path))
        except Exception as exc:
            context.registry.mark_error(
                self.name,
                f"failed to load token: {type(exc).__name__}: {exc}",
            )
            return

        if getattr(creds, "expired", False):
            try:
                creds.refresh(Request())
            except Exception as exc:
                context.registry.mark_error(
                    self.name,
                    f"token refresh failed: {type(exc).__name__}: {exc}",
                )
                return

        self._creds = creds
        context.tool_registry.register(_list_events_tool(self), replace=True)
        if os.environ.get(WRITE_ALLOWED_ENV) == "true":
            context.tool_registry.register(_create_event_tool(self), replace=True)
        context.registry.mark_running(self.name)
```

`replace=True` lets the adapter re-bind cleanly in tests
without raising on duplicate registration.

## Tool factories

`_list_events_tool` and `_create_event_tool` follow the
`read_file_tool` factory pattern. They close over the adapter
so handlers can reach `self._creds` and `self._context`.

```
def _list_events_tool(adapter: "CalendarAdapter") -> Tool:
    def handler(calendar_id: str = "primary") -> list[dict[str, Any]]:
        from googleapiclient.discovery import build

        ctx = adapter._context
        try:
            start_iso, end_iso = _today_utc_window()
            service = build("calendar", "v3", credentials=adapter._creds)
            response = (
                service.events()
                .list(
                    calendarId=calendar_id,
                    timeMin=start_iso,
                    timeMax=end_iso,
                    singleEvents=True,
                    orderBy="startTime",
                )
                .execute()
            )
            items = response.get("items", [])
            events = [_serialize_event(item) for item in items]
            if ctx is not None:
                ctx.registry.touch(adapter.name)
            return events
        except Exception as exc:
            if ctx is not None:
                ctx.registry.mark_error(adapter.name, f"{type(exc).__name__}: {exc}")
            return [{"error": f"{type(exc).__name__}: {exc}"}]

    return Tool(
        name="list_calendar_events_today",
        description="List today's events from the configured Google Calendar (UTC window).",
        parameters={
            "type": "object",
            "properties": {
                "calendar_id": {
                    "type": "string",
                    "description": "Google Calendar ID; defaults to the user's primary calendar.",
                },
            },
            "required": [],
        },
        handler=handler,
    )


def _create_event_tool(adapter: "CalendarAdapter") -> Tool:
    def handler(
        summary: str,
        start_iso: str,
        end_iso: str,
        calendar_id: str = "primary",
        description: str | None = None,
    ) -> dict[str, Any]:
        if os.environ.get(WRITE_ALLOWED_ENV) != "true":
            return {
                "error": (
                    "calendar write not allowed; set "
                    f"{WRITE_ALLOWED_ENV}=true to enable"
                )
            }
        from googleapiclient.discovery import build

        ctx = adapter._context
        try:
            service = build("calendar", "v3", credentials=adapter._creds)
            body: dict[str, Any] = {
                "summary": summary,
                "start": {"dateTime": start_iso},
                "end": {"dateTime": end_iso},
            }
            if description is not None:
                body["description"] = description
            response = (
                service.events()
                .insert(calendarId=calendar_id, body=body)
                .execute()
            )
            if ctx is not None:
                ctx.registry.touch(adapter.name)
            return response
        except Exception as exc:
            if ctx is not None:
                ctx.registry.mark_error(adapter.name, f"{type(exc).__name__}: {exc}")
            return {"error": f"{type(exc).__name__}: {exc}"}

    return Tool(
        name="create_calendar_event",
        description=(
            "Create a single event on the configured Google Calendar. "
            "Requires HORUS_OS_CALENDAR_WRITE_ALLOWED=true."
        ),
        parameters={
            "type": "object",
            "properties": {
                "summary": {"type": "string", "description": "Event title."},
                "start_iso": {
                    "type": "string",
                    "description": "Event start time in ISO 8601 (e.g. 2026-05-24T15:00:00-07:00).",
                },
                "end_iso": {
                    "type": "string",
                    "description": "Event end time in ISO 8601.",
                },
                "calendar_id": {
                    "type": "string",
                    "description": "Calendar ID; defaults to the user's primary calendar.",
                },
                "description": {
                    "type": "string",
                    "description": "Optional event description.",
                },
            },
            "required": ["summary", "start_iso", "end_iso"],
        },
        handler=handler,
    )
```

## Helpers

```
def _today_utc_window() -> tuple[str, str]:
    now = datetime.now(UTC)
    start = now.replace(hour=0, minute=0, second=0, microsecond=0)
    end = now.replace(hour=23, minute=59, second=59, microsecond=0)
    return start.isoformat(), end.isoformat()


def _serialize_event(item: dict[str, Any]) -> dict[str, Any]:
    start = item.get("start", {}) or {}
    end = item.get("end", {}) or {}
    attendees = [
        att.get("email", "") for att in (item.get("attendees") or [])
    ]
    return {
        "summary": item.get("summary", ""),
        "start": start.get("dateTime") or start.get("date", ""),
        "end": end.get("dateTime") or end.get("date", ""),
        "location": item.get("location", ""),
        "attendees": attendees,
    }
```

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| OAuth token file on disk -> `Credentials.from_authorized_user_file` | A leaked token file grants the configured Calendar scope to anyone who can read it |
| `HORUS_OS_CALENDAR_OAUTH_CLIENT_PATH` env -> Credentials | Client secret JSON exposes the OAuth app to abuse; less sensitive than the token but still a secret |
| `HORUS_OS_CALENDAR_WRITE_ALLOWED` env -> `events().insert` | A misconfigured env var grants the agent the ability to create calendar events |
| Agent tool call -> Google Calendar API | A compromised or prompt-injected agent can read or create events on the user's calendar |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-26-01 | Information Disclosure | Token JSON readable on disk by other local users | mitigate | The setup guide documents that the token must live under the user's `data_dir` with restrictive permissions; the wizard helper that creates the data dir already uses 0700-style perms on POSIX |
| T-26-02 | Tampering | Prompt injection causes the agent to leak event contents | accept | The agent owns the response; tool returns plain dicts. Per-tool gating is the v0.4 concern (Phase 27 dashboard view exposes tool inventory) |
| T-26-03 | Elevation of Privilege | Agent creates events without operator intent | mitigate | `HORUS_OS_CALENDAR_WRITE_ALLOWED` is opt-in (must be exactly `"true"`); the tool is not registered at all when the flag is unset, and the handler re-checks the env at call time |
| T-26-04 | Denial of Service | Agent loops on a calendar tool that always errors | accept | Tool returns `{"error": ...}` rather than raising; agent loops are bounded by the existing turn count cap in `run_agent_loop` |
| T-26-05 | Repudiation | No audit trail of which agent invocation created which calendar event | accept | The trace table already records every tool_use the agent emits; the adapter relies on that surface. Phase 27 surfaces this in the UI |
| T-26-06 | Spoofing | Adversary swaps the token file for one belonging to another account | accept | Local file system trust; same boundary as the database file itself |
</threat_model>
