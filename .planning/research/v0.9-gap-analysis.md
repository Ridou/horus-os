# v0.9 Feature-Gap Analysis

Generated 2026-06-03. Source comparison: the mature private `horus` codebase,
the open-source Paperclip product (via prior distilled research), and the
open-source Odysseus AI workspace. Scope filter: generic agent-platform
capabilities only. Personal-life features (contacts, photos, health, travel,
yoga, people graph) are excluded. The private PR-review pipeline is excluded by
project rule. Private modules are referenced by bare filename only.

This document is a landscape, not a plan. It catalogs what horus-os is missing
relative to a much older sibling codebase, so v0.9 scope can be chosen
deliberately. There is far more here than one milestone.

---

## Theme A: Memory intelligence

horus-os today has a basic markdown vault (read + write) plus an ONNX +
sqlite-vec KNN index. It has no intelligence layer on top of that store. This is
the single deepest gap area.

| Gap | What it adds | Value / Effort |
|-----|--------------|----------------|
| Structured fact extraction + conflict resolution + version chains (`memory_engine.py`) | Extract entity-centric facts per task; dedup at >=0.90 similarity, LLM-classify contradict/extend at >=0.85, supersede with preserved version history. The leap from "vector store" to "memory engine." | High / M |
| Auto-memorization (`auto_memorize.py`) | Fire-and-forget post-task extraction of 1-5 fragments, two-gate consolidation, daily caps, secret masking. Turns memory from manual to ambient. | High / M |
| Dream / sleep consolidation (`dream_consolidation.py`, `dream.py`) | Gated background pass (time + session gates, crash-safe lock) that consolidates and prunes memory; full engine adds Hebbian linking and retrieval-reversal. | High / M (light) - L (full) |
| Fragment consolidation sweep (`memory_consolidation.py`) | Nightly merge of old same-category fragments into summaries; prevents unbounded growth. ~320 lines, one hook. | High / S |
| Working-memory tier (`working_memory.py`) | Read-only top-k prefetch over recent traces into a sanitized prompt block. Short-term recall the system lacks. | Med-High / S |
| Relevance-ranked recall + staleness caveats (`memory_recall.py`) | LLM picks top 3-7 memories instead of dumping all; stamps age warnings; strips prior memory tags to stop accumulation across turns. | Med-High / S-M |
| Intelligent forgetting (`memory_engine.py`) | Temporal expiry (`forget_after`) + explicit soft-delete with reason. | Med / S |
| Session-context handoff (`session_context.py`, `history_compress.py`) | Structured 9-section cross-session summary; long-session history compression that pins the active task. | Med / S-M |
| Correlation engine (`correlation_engine.py`) | Mine causal A->B->C chains across task outcomes via temporal + semantic + LLM-plausibility filter. | Med / M |
| Vault drift scanner (`vault_drift.py`, `librarian.py`) | Flag stale / orphan / incomplete notes; knowledge-debt metric. Cheap observability over the vault. | Med / S |

Design note: the whole memory layer routes through the cheapest model tier with
daily caps and fire-and-forget semantics (~$0.15-0.30/month). horus-os already
has a cheap-model concept and a vector index, so this layers on without new
infra. Biggest single multiplier is structured facts; recall/forgetting/history
all build on the fact model.

horus-os has delegation plus one hardcoded Deep Research workflow.

| Gap | What it adds | Value / Effort |
|-----|--------------|----------------|
| Reusable workflow/chain engine (`chains.py`, `workflows.py`) | Multi-step chains as a checkpointed state machine with resume, inter-step result forwarding, templates, aggregation strategies, stale-chain recovery. Turns delegation into composable restartable pipelines. | High / M-H |
| Capability router (`capability_router.py`) | Keyword-score a task to the tool manifest it needs; strictly additive (never strips baseline tools). Zero-latency, no LLM. | High / L-M |
| Complexity estimation + model-tier gating (`complexity.py`) | Label simple/medium/complex via heuristics + optional cheap-LLM fallback; route simple work to cheap models, trigger chains for complex. ~150 lines. | High / L |
| Spontaneous / idle autonomy (`spontaneous.py`, `synapse.py`) | Self-initiated tasks in idle windows behind a strict gate chain (idle check, refractory cooldown, per-agent + daily USD cap, jitter, anti-runaway-feedback). The headline "autonomous" capability. | High / M-H |
| Overnight batch runner (`overnight.py`) | Nightly cycle: allow-list + opt-in + mandatory pre-execution disclosure, single attempt, morning report. | High / M |
| Context-handoff generator (`handoff_generator.py`) | 9-section handoff when a long task hits context limit, so a fresh session resumes. ~160 lines. | Med-High / L |
| Priority execution lanes (`lanes.py`) | Per-lane semaphores (MAIN / CRON / SUBAGENT / OVERNIGHT) so background work never starves interactive work. Safety substrate for autonomy. | Med-High / L |
| Idle backlog drainer with preemption (`ralph.py`) | Drain a markdown backlog only when idle; cancel + re-queue background work the moment a user task arrives. | Med / L-M |

chain engine, with lanes as the substrate that lets spontaneous + overnight
autonomy run without starving user traffic.

## Theme D: Cost control, approval, and safety

horus-os measures cost but does not enforce it; it has no human-approval gate and
no output redaction. Given v0.8 shipped autonomous research and gated shell
execution, this is arguably the most responsible next step.

| Gap | What it adds | Value / Effort |
|-----|--------------|----------------|
| Monetary budget enforcement (`budget_enforcer.py`) | Budgets table (agent/project/global, soft+hard thresholds); post-run check pauses the agent and records an incident on hard breach. horus-os already persists per-call cost. | High / M |
| Daily cost ceiling pre-flight guard (`cost_guard.py`) | Block a run from starting when today's spend exceeds a configurable cap. ~40 lines + one config key. | High / L |
| Tiered smart approval (`smart_approval.py`) | Risk-classify tool calls AUTO/REVIEW/CONFIRM; learn from history (3+ approvals -> auto, 2+ denials -> confirm); high-risk always confirms. | High / M |
| Secrets-redaction filter (`secrets_filter.py`) | Central masking engine (format patterns for common key shapes + user patterns) wrapping every output/log path. Defensive for a tool that runs shell + web fetch. | High / L-M |
| Destructive-action approval gate (`approval.py`) | Scan prompts for destructive patterns; gate autonomous (not user) tasks pending approval; info-only classifier bypasses. Pairs with smart approval. | Med / M |
| Subprocess resource guard (`subprocess_mgr.py`) | Memory cap + recursive tree-kill + timeout + orphan reaping for spawned processes. Generalizes to shell_exec. | Med-High / M |
| Skill threat scanner + integrity registry (`skill_safety.py`) | SHA256-pin trusted skills (tamper block) + ~100 regex threat patterns scanned on skill prompts. Relevant for user-authored skills/plugins. | Med-High / M |
| Verification gate (`verification.py`) | Confidence-scored pass/fail/revise self-verification with escalate-to-human threshold. Strong fit for autonomous research output. | Med / M |
| Stale-run reaper (`process_reaper.py`) | Mark tasks stuck `running` past a cutoff as failed so a crashed worker does not wedge the queue. ~15 lines on a tick. | Med / L |

Natural pairings: pre-flight ceiling + post-run budget pause as one cost-control
feature; smart approval + destructive gate as one risk-tiered approval system
with a shared classifier.

## Candidate v0.9 shapes

There is too much here for one milestone. The themes suggest three coherent
narratives, each on-brand for an "autonomous AI command center":

1. **Autonomy and Control** (Themes C + D + E). Make agents act on their own,
   safely. Event bus + watches/triggers + heartbeat/spontaneous loop + lanes,
   gated by budget enforcement + tiered approvals + secrets filter + verification,
   surfaced through an approvals queue + unified inbox + run-liveness. Reuses the
   most existing infra (cost metering, scheduler, multi-agent). Multiple agents
   independently flagged these as the highest-value, lowest-effort wins, and it is
   the responsible follow-up to v0.8 shipping autonomy + shell exec with no spend
   cap or approval gate.

2. **Memory and Learning** (Themes A + B). The structured memory engine plus the
   self-improvement loop. The deepest differentiator and the truest meaning of
   "agents that get better," but heavier and less immediately visible.

3. **Work Legibility** (Theme F + parts of E). The issue-system and dashboard
   upgrade: detail page, kanban, run-liveness, references, command palette,
   approvals + inbox UI. The most demoable and Paperclip-aligned, and a natural
   home for surfaces the other two themes produce.

Recommendation: lead v0.9 with **Autonomy and Control**, because it converts
v0.8's raw capability into safe, self-directed operation and closes a real
safety gap (metered-but-unbounded spend, no human-in-the-loop gate). It pulls a
thin slice of Theme F with it (approvals queue, inbox, run-liveness) and sets up
Memory and Learning as a strong v0.10.
