## What only YOU can do (rehearsal session checklist)

### A. The v0.6.0-rc1 release rehearsal (about 30-60 min, closes 5 UAT items)

1. On a fork: build wheel + sdist (`python -m build`).
2. `gh release create v0.6.0-rc1 --title "rc1 rehearsal" --notes "..."` on the fork.
3. Wait for `release.yml` to run end-to-end. Capture:
   - `<wheel>.whl.sigstore.json` plus `tests/fixtures/sigstore/canonical/`
   - The two SBOMs plus their `.sigstore.json` files in the same dir
4. `gh attestation verify <wheel> --repo <fork>` exits 0
5. `gh attestation verify <sbom>.cdx.json --repo <fork>` exits 0
6. Run `git apply --check .planning/rollback/flip-gate-revert.md` against a
   stale tree, exits 0 (RUNBOOK-02 UAT item)
7. Commit the fixtures:
   `git add tests/fixtures/sigstore/canonical/ && git commit -m "test(58): commit canonical sigstore fixtures from v0.6.0-rc1 rehearsal"`
8. Flip the 2 SKIPped canonical-fixture tests to GREEN:
   `pytest tests/test_release_verification.py::test_canonical_fixture_passes_wheel_check -v`

### B. Gitsign workstation config (about 5 min, closes UAT item 6)

```
brew install sigstore/tap/gitsign      # or wget the release binary
git config --global commit.gpgsign true
git config --global gpg.x509.program gitsign
git config --global gpg.format x509
git config --global gitsign.connectorID https://github.com/login/oauth
git config --get gitsign.connectorID   # confirm non-empty
git tag -s v0.6.0-test -m "test"        # opens browser for OAuth
git verify-tag v0.6.0-test              # exit 0
git tag -d v0.6.0-test                   # cleanup
```

## Rollback plan

`.planning/rollback/flip-gate-revert.md` documents three rollback methods
(git revert preferred, hard reset destructive, patch apply for the
already-pushed case). Steps to roll back:

1. `git revert <FLIP_SHA>` (preserves history, downstream-safe)
2. If the tag pushed: `git push origin :refs/tags/v0.6.0`
3. Delete the GitHub Release: `gh release delete v0.6.0 --yes`
4. STATUS.md flips back to "solo development mode" within the revert
   commit
5. Announce on the pinned "Project Status" Discussion why the flip was
   backed out and when the next attempt will land
6. Update `.planning/STATE.md` to reflect v0.6 is NOT shipped
