# Rollback: v0.6.0 contribution-gate flip

If the Phase 59 atomic gate-flip commit needs to be backed out (CI regression, contributor-facing copy bug, or any reason the maintainer wants to walk it back within 24 hours of the flip), apply this template.

The flip commit changes these files in a single atomic commit:

- `STATUS.md` TL;DR + milestone row
- `README.md` "Project status" section + badge
- `CONTRIBUTING.md` "Status: not currently accepting" block deleted; "Future contribution flow (not active yet)" deleted; PHASE-59-FLIP markers removed
- `.github/PULL_REQUEST_TEMPLATE.md` NOTICE block deleted
- `.github/ISSUE_TEMPLATE/bug.yml` + `feature.yml` PHASE-59-FLIP banners removed
- `SECURITY.md` "Contributor-pipeline security (not active yet)" section deleted; PHASE-59-FLIP marker removed
- `.github/workflows/issue-claim-watcher.yml` deleted
- Saved replies updated for the post-flip claim flow
- `CHANGELOG.md` `[Unreleased]` promoted to `[0.6.0]`
- New `.github/workflows/` settings if any

## Rollback method 1: git revert (preferred)

If the flip commit is the most recent commit on `main` and nothing has been pushed downstream:

```bash
# Find the flip commit SHA
FLIP_SHA=$(git log --oneline --grep="^chore(59):" -1 --format=%H)

# Revert it (creates a new commit that undoes the flip)
git revert --no-edit "${FLIP_SHA}"

# Push the revert
git push origin main
```

This is the cleanest rollback because it preserves the history of both the flip and the revert. Downstream consumers (anyone who pulled the flip) can fast-forward to the revert without re-cloning.

## Rollback method 2: git reset --hard (use with care)

ONLY if the flip commit was pushed less than 30 minutes ago AND no downstream contributors have pulled yet. This rewrites history; coordinate via the Discussion post before doing this.

```bash
# Find the SHA of the commit BEFORE the flip
PRE_FLIP_SHA=$(git log --oneline --grep="^chore(59):" -1 --format=%H^)

# Hard reset
git reset --hard "${PRE_FLIP_SHA}"

# Force push (DESTRUCTIVE; coordinate first)
git push --force-with-lease origin main
```

## Rollback method 3: literal patch apply

If the flip commit cannot be reverted automatically (merge conflict, partial revert needed, the maintainer wants the rollback to be a hand-crafted commit instead of a git revert artifact), apply the following patch. Test it with `git apply --check < patch.diff` first.

The patch contents are auto-generated by Phase 58 rehearsal: at rehearsal time, the maintainer runs:

```bash
# Generate the inverse patch from the planned flip
git diff "${PRE_FLIP_SHA}".."${FLIP_SHA}" -- \
  STATUS.md README.md CONTRIBUTING.md \
  .github/PULL_REQUEST_TEMPLATE.md \
  .github/ISSUE_TEMPLATE/bug.yml \
  .github/ISSUE_TEMPLATE/feature.yml \
  SECURITY.md \
  > .planning/rollback/flip-gate-revert.patch
```

Then to apply the rollback:

```bash
# Apply the inverse patch
git apply -R .planning/rollback/flip-gate-revert.patch

# Restore the deleted issue-claim-watcher.yml from the pre-flip SHA
git checkout "${PRE_FLIP_SHA}" -- .github/workflows/issue-claim-watcher.yml

# Stage + commit the rollback
git add STATUS.md README.md CONTRIBUTING.md \
        .github/PULL_REQUEST_TEMPLATE.md \
        .github/ISSUE_TEMPLATE/bug.yml \
        .github/ISSUE_TEMPLATE/feature.yml \
        SECURITY.md \
        .github/workflows/issue-claim-watcher.yml

git commit -m "chore(rollback): revert v0.6.0 contribution-gate flip

Reason: <fill in>
Phase 59 atomic flip commit: <FLIP_SHA>
"
```

## After rollback

1. Announce on the pinned "Project Status" Discussion post why the flip was backed out and when the next attempt will land.
2. Open a tracking issue with `type:bug` + `area:cli` (or the appropriate area label) capturing the specific failure that triggered the rollback.
3. Update `.planning/STATE.md` to reflect that v0.6 milestone is NOT shipped.
4. Add a note in `CHANGELOG.md` `[0.6.0]` section explaining the deferral and reverting any `0.6.0` promotion to `[Unreleased]`.
5. Tag and push a `v0.6.0-postmortem` tag pointing to the rollback commit so the rollback itself is auditable.

## Phase 58 rehearsal note

This rollback template is `git apply`-tested as part of the Phase 58 release rehearsal. The test asserts:

- `git apply --check` against a stale working tree (a fork at the Phase 56 head) exits 0.
- After applying, the working tree matches the pre-flip state file-for-file.
- The test catches the case where Phase 59's atomic flip introduces a new file the rollback patch does not know about.

If the Phase 58 rehearsal flags a missing file in the rollback, update this template before v0.6.0 ships.
