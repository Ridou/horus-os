## Our standard

`horus-os` adopts the [Contributor Covenant version 2.1](https://www.contributor-covenant.org/version/2/1/code_of_conduct/) as its code of conduct. It applies to everyone who participates in the project, whether you are filing an issue, opening a pull request, joining a discussion, or interacting in any other project space.

The full text of Contributor Covenant 2.1 is available at:

https://www.contributor-covenant.org/version/2/1/code_of_conduct/

## Reporting

The project is solo-maintained and does not yet publish a dedicated conduct contact address. Until one is published, report a conduct concern privately by opening a draft GitHub Security Advisory at https://github.com/Ridou/horus-os/security/advisories/new. This channel is used here as the project's private contact channel.

When you file a report through that channel, note in the report that it is a conduct matter, not a security vulnerability, so it is routed correctly.

For non-sensitive matters, you can also open a [GitHub Discussion](https://github.com/Ridou/horus-os/discussions).

> [!NOTE]
> The security advisory form is reused as the private contact channel only because it is the project's existing confidential inbox. A conduct report is not a vulnerability disclosure, so always state clearly that your report concerns conduct.

## Enforcement

The project maintainers commit to responding to any reported violation within seven days and applying the Contributor Covenant enforcement ladder. The enforcement ladder, including the categories of impact and the corresponding consequences, is described in full in the linked Contributor Covenant 2.1 text.
